## 運営メモ（アンケートには含めない）

### Valueの位置づけ

今回の表彰を機に、mTG MVVのValueをデジタルプラットフォームＧ全体の行動指針として位置づける。キックオフで宣言する。

背景：
- Valueの3項目（お客さま起点・圧倒的当事者意識・感謝と尊敬）はmTG固有ではなく、TGK・EA含むグループ全体の仕事に当てはまる内容
- SDチーム（片山TL）もTGK担当メンバー含めMVVを意識して業務している
- 対象をmTGに限定すると、TGK・EAメンバーのモチベーションにマイナスに働くリスクがある

### 投票対象・投票者

| 項目 | 内容 |
|---|---|
| 投票対象 | デジタルプラットフォームＧ 全メンバー（TGK担当・EA推進含む） |
| 投票者 | 同上（自分自身への投票は不可） |
| 異動者 | FY2025年度中に当グループで活動していた方は対象に含める |
| 匿名性 | 集計者（杉﨑）には回答者が見える。発表時には「寄せられたコメント」として匿名で共有 |

### 集計・選考

| 項目 | 内容 |
|---|---|
| 集計方法 | Valueごとに得票数とコメント内容を整理 |
| 選考方法 | **投票結果（票数＋コメント）を材料に、GM＋TL合議で受賞者を決定**。最多得票で自動決定しない |
| 選考の観点 | コメントの質・具体性を重視する。票が少なくてもコメントの説得力が高ければ候補になる |
| 同票の場合 | コメントの質を加味してTL合議で決定 |
| 発表方法 | 受賞者の名前 ＋ 寄せられた一言コメント（匿名）をまとめて共有 |
| 発表の場 | キックオフ（グループ全体の取り組みとして位置づけ、mTG限定の文脈に置かない） |

### 設計上のポイント

- **Value原文をそのまま掲載**する。抽象化せず、読んだ上で「この人だ」と思える状態にする
- **理由の一言コメントが最も重要**。票数だけだと人気投票になるが、コメントがあることで「なぜその人か」が言語化され、表彰の説得力が上がる
- **票数で自動決定しない**。人数比の大きいmTG系メンバーに票が集中しやすい構造のため、TL合議で公平性を担保する
- 発表時にコメントを本人に届けることで、**表彰が「周囲からのフィードバック」として機能する**
