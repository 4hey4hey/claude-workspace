## このチームは何のために存在するか

### Mission

> **バリューチェーンの流れを最適化し、ムダなくムリなく顧客に価値が届く状態を作る。**
> 

経営にとっての意味は「同じ予算体制で、施策をもっと速く・多く回せるようにする」こと。

### Missionの構造

Missionは3つの柱で構成される。

```
Mission：開発投資のリターンを最大化する
│
├─ 投資対効果（正しいものに正しい順番で投資する）
│    行動原則「見極める」← PO機能
│
├─ スループット（速く届ける）
│    行動原則「速くする」← SM + PO
│
└─ サステナビリティ（走り続けられる）
     行動原則「守る」「強くする」← SM+PO
```

---

### 行動原則

| **原則** | **担い手** | **意味** | **具体的な場面** |
| --- | --- | --- | --- |
| 👁 **見極める** | PO | 投資判断・価値の選択 | 優先順位判断、やらないことの決定、受入条件定義、スコープ判断、レビューでの価値検証 |
| ⚡ **速くする** | SM＋PO | フロー効率の最大化 | ボトルネック解消、PBIの小分割、仕様起因の手戻り削減、リードタイム短縮 |
| 💪 **強くする** | SM+PO | チーム育成・プロセス改善 | チームの育成、オンボーディング、ふりかえりの質向上、スクラム標準化 |
| 🛡 **守る** | SM | 障害除去・健全性維持 | ブロッカー検知・除去、1on1、メンタルケア、負荷分散 |

## 📊 杉崎のOKR

### Objective（目指す姿）

> **投資が最短・最確実に顧客価値へ変換されるデリバリーラインを確立する**
> 

「速く作る」だけではありません。**「正しいものを、速く、持続的に届ける」**状態を作ることです。

---

### 5つのKR（重要成果指標）

| **KR** | **何を目指すか** |  |
| --- | --- | --- |
| 🔴 **KR1**
入口品質 | 開発ラインに流す仕事の質を上げ、仕様起因の手戻りをなくす |  |
| 🔴 **KR2**
フロー効率 | 着手からリリースまでの時間を短縮する |  |
| 🔴 **KR3**
学習ループ | 届けた結果から学び、次の投資判断に活かすサイクルを作る |  |
| 🔵 **基盤KR1**
健全性 | チームが持続可能なペースで走れる状態を守る |  |
| 🔵 **基盤KR2**
育成 | 次世代リーダーを育てる |  |

---

### 3つの挑戦KRはつながっている

<aside>
🔄

**入口品質（KR1）→ フロー効率（KR2）→ 学習ループ（KR3）** の順に連動して改善されます。

- KR1が改善 → 手戻りが減り、フロー（KR2）が速くなる
- KR2が改善 → 速く届く → 検証頻度が上がり、学習（KR3）が回る
- KR3が回る → 学びが仮説精度を高め、入口品質（KR1）がさらによくなる

**どれか1つだけやっても効果は限定的。3つが連動して初めてMissionが達成されます。**

基盤KR1・KR2は、この3つすべての前提条件です。

</aside>

---

### POとSMを貫くひとつの原則

> **価値を小さく分割し、比較可能な単位で届け、仮説検証のスピードを上げる**
> 

| **実践** | **やること** | **効くKR** |
| --- | --- | --- |
| 価値の垂直スライス | 技術レイヤーではなく、ユーザー価値の単位で分割する | KR2・KR3 |
| 比較可能な単位で出す | A案/B案を同時に出して定量比較できるサイズにする | KR3→KR1 |
| 仮説を明示してから着手する | PBIに「検証したい仮説」を1行書いてスプリントに入れる | KR3・KR1 |
| 「やらない」を先に決める | 分割時に今回やらないことを明示的にリストアップする | KR1・KR2 |

### Mission→OKR

Missionを「問いの形」でロジックツリーに分解し、KRを導出した。

```
【問いの形】
開発投資のリターンを最大化するために、デリバリーチームは何を実現すべきか？
│
├─ 開発ラインに流す仕事の質と判断スピードをどう高め、仕様起因の手戻りと無駄な開発をなくせるか？
│   （入口品質 / 見極める）
│   ├─ バックログを常にReadyに保ち、曖昧な仕様のまま着手させないためには？
│   ├─ 仕様起因の手戻りをどう根絶するか？
│   ├─ PO判断がフローを止めないようにするには？
│   ├─ 関連プロジェクトの依存関係や影響度合いなどを透明性をあげるには？
│   └─ スコープ膨張をどう防ぎ、やらない判断をどう徹底するか？
│
├─ 届けた価値を検証し、学びを次のサイクルにどう反映し続けるか？
│   （学習ループ / 見極める）
│   ├─ スプリントレビューで価値仮説を確実に検証するには？
│   ├─ 検証結果をPdMにフィードバックし、次の投資判断に反映するには？
│   └─ 「作った→検証→学んだ→次の優先順位に反映」のサイクルをどう定着させるか？
│
├─ 顧客に届くスピードをどう上げるか？（フロー効率 / 速くする）
│   ├─ サイクルタイムをどう短縮するか？
│   ├─ ボトルネックを可視化し渋滞をどう解消するか？
│   └─ スプリントゴールを確実に達成するには？
│
└─ チームが走り続けられる力をどう担保するか？（サステナビリティ / 守る・強くする）
    ├─ メンバーが健全なペースで働けているか？
    └─ 次世代リーダーをどう育てるか？
```

> **KR間の因果接続：** KR1（入口）→ KR2（フロー）→ KR3（出口・学習）で開発バリューチェーンの全区間をカバーする。KR3の学習結果がKR1の入口品質をさらに改善する循環構造。
> 

### Objective

> **バリューチェーンを「投資判断→開発→顧客価値」の一気通貫で回し、プロダクト投資が最短・最確実に顧客価値へ変換されるデリバリーラインを確立する**
> 

Missionの3つの柱（投資効果・スループット・サステナビリティ）を統合したObjective。単に速く作るだけでなく、「正しいものを、速く、持続的に届ける」ことを目指す。

### ロジックツリー

```
O: 投資が最短・最確実に顧客価値へ変換されるデリバリーラインを確立する
│
├─ 挑戦KR1: 開発ラインの入口品質を確立し、仕様起因の手戻りコストを
│   2028年度末までに限りなくゼロに近づける（入口品質 / 見極める）
│
├─ 挑戦KR2: サイクルタイムを2028年度末までに現状比50%短縮する
│   （フロー効率 / 速くする）
│
├─ 挑戦KR3: 「届けて→検証→学ぶ→次に活かす」投資判断の学習サイクルを
│   2028年度末までに全チームで定着させる（学習ループ / 見極める）
│
└─ 基盤KR: チーム健全性と育成基盤を確立する（サステナビリティ）
    ├─ チーム健全性スコアを全チーム「良好」以上で維持する（通年）
    └─ チーフ育成計画のマイルストーン達成率100%を実現する（2026年度末）
```
