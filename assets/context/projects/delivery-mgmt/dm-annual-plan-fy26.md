## 年度方針

### 一言で言うと

> **価値提供施策の打席数を増やし、打席の精度を上げる。**

### FY26の位置づけ

FY26は「計測と構造整備」の年。
「開発が遅い」の正体は、入口品質・フローの未安定・学習ループの不在という3つの構造的課題。
この3つに同時に手を打つための土台を作り、Q1末に具体的な数値目標をセットする。

```text
FY26（本年度）         FY27                FY28
計測＋構造整備     →  改善サイクル定着    →  自律運転
```

### FY26末の到達イメージ

| 領域 | FY26末の状態 |
|---|---|
| 入口品質（KR1） | 全チームでReady定義が標準化。手戻り原因分類が運用され、仕様起因手戻りが削減傾向 |
| フロー効率（KR2） | サイクルタイム計測が全チームで稼働。未安定3チームのCTがNaikistレベルに近づいている |
| 学習ループ（KR3） | スプリントレビューに価値検証アジェンダが定着。学習ログ蓄積＋PdMへの月次FB稼働 |
| 健全性（基盤KR1） | 全チーム健全性スコア「良好」以上を通年維持 |
| 育成（基盤KR2） | チーフ育成計画マイルストーン100%達成。PO候補2名（伊藤さん・室山さん）の育成進捗 |

> **注：** 方向指標（手戻り率・スプリントゴール達成率・開発配分・サイクルタイム）の具体的な数値目標はQ1末に計測結果をもとにセットし、及川GMに報告する。

### DPG OKRとの紐づけ

```text
【DPG Objective】
  SoE領域のEAを整備し、mTGを中核に
  「使えば使うほど暮らしと地球がよくなる日本一のデジタルPF」を実現する
  │
  ├─ 挑戦KR① 熱狂顧客の獲得  ←── PdMチーム主管
  │
  ├─★挑戦KR② 高速プロダクト開発の型 ←── デリバリーチームが直接担う
  │       │
  │       ▼
  │   【デリバリーチーム Objective】
  │       ├─ 挑戦KR1: 入口品質（仕様起因手戻り→ゼロへ）
  │       ├─ 挑戦KR2: フロー効率（サイクルタイム50%短縮）
  │       ├─ 挑戦KR3: 学習ループ（検証→次の投資判断へ）
  │       ├─ 基盤KR1: チーム健全性維持
  │       └─ 基盤KR2: 次期リーダー育成
  │
  ├─ 基盤KR③ EA関連PJの着実な進捗 ←── 各PJチーム
  ├─ 基盤KR⑥ 運営コスト削減        ←── デリバリーのフロー改善も間接貢献
  └─ 基盤KR⑦ ビジネス側評価        ←── デリバリーの品質・スピードが直接影響
```
