# myTOKYOGAS（B2Cサービス）

## メタ情報

| 項目 | 内容 |
|---|---|
| **フェーズ** | 運用中（継続的なエンハンス開発 + 3ヶ年ロードマップ策定中） |
| **担当チーム** | Naikist（Webフロント）、Fluppuccino（モバイルアプリ）、Guardians（マイクロサービス）、SAGAS（クラウドPF）、コンテンツチーム。外注：iネット（OPST、TDC等） |
| **主要ステークホルダー** | 及川GM、大橋部長（ロードマップ承認者） |

## 概要

東京ガスのガス・電気利用者向けWeb会員サービス。毎月の請求金額・使用量照会、パッチョポイント、各種お手続き、お役立ち情報を提供するB2C顧客接点基盤。2003年開始、2016年電力自由化対応でフルリニューアル、2019年にクラウド移行（mTG2.0）を経て現在に至る。

## 主要機能

| 機能 | 概要 |
|---|---|
| **料金・使用量照会** | ガス／電気／域外の月別料金・使用量を表示。PDF・CSV出力対応 |
| **パッチョポイント** | 利用額に応じたポイント付与（1.5%）。提携先ポイントへの交換・寄付が可能 |
| **マイページ** | 契約情報・会員情報の確認・変更、メール配信設定、機器情報管理 |
| **キャンペーン** | バナー・告知によるキャンペーン運用 |
| **検針票電子化** | 2023年11月稼働。紙の検針票に代わる情報をWeb上で提供。2024年10月に紙投函終了・有償化 |

## システム構成

| コンポーネント | 技術・サービス |
|---|---|

| **CMS** | WebRelease2（WR2）— お知らせ・コンテンツ管理 |

| **関連システム** | CIRIUS（ガスCIS）、電力CIS、TG-CRM、TG-MA（マーケティングオートメーション）、LINE連携 |
| **インフラ** | 一部Oracle Cloud、BFFアーキテクチャ |

## 会員体系

### 会員種別

| 種別 | 説明 |
|---|---|
| **本会員** | ID/PW設定済みでログイン可能 |
| **仮登録** | メールによる自動登録後、未ログイン状態 |
| **未会員** | 契約はあるがmTG未登録 |
| **ライト会員** | 限定的な機能のみ利用可能 |
| **プレ会員** | 契約前の仮登録状態 |

### 登録フロー

- **能動登録**: お客さま自身がWebから新規登録
- **自動登録（メール）**: Web引越し・契約受付時にメールで案内
- **自動登録（ハガキ）**: 電話・書面受付時にハガキで初期ID/PW通知

### 契約種別

ガス契約、電気契約、域外電気契約、くらしサービス契約

## セキュリティ・認証

### 情報表示ランク

| ランク | 対象 | 保護方針 |
|---|---|---|
| **SS** | パスワード、秘密の質問回答 | 表示不可。ハッシュ化保持 |
| **S** | 住所・氏名・電話番号・メアド・支払方法・ポイント交換 | ID/PW＋二段階認証（メールPINまたは電話認証）が必要 |
| **A** | 上記以外（お客さま番号・使用量・請求額など） | ID/PWのみで表示可 |

### 認証フロー

1. **ログイン**: IDCS（ID/PW）→ 任意でSMS/メール二段階
2. **個人情報閲覧・変更**: 二段階認証（メールPIN）または電話認証
3. **ポイント交換**: 電話認証 → 24時間猶予 → 電話認証パトロール
4. **パスワード再設定**: 登録メアドへ再設定メール → PINコード認証
5. **初期ID確認**: 契約情報（住所・氏名・電話番号）で特定 → 電話認証

## mTG ↔ Kraken 直連携

> Kraken Migration PJ 本体は `projects/kraken/` を参照。ここでは mTG 側から見た直連携の課題を記述する。

**概要：** mTGのバックエンドを、CIRIUSベースからKrakenへ直接連携させるサブPJ。CIRIUSからの発番依存を解消し、Krakenから直接料金・契約情報を取得できる構造を目指す。

**リリース計画：** 26年度から27年度にかけて3段階リリース予定
1. 事業法の必要機能を実装して新料金リリースに適応（26年度中盤）
2. 現在のmTG同等の機能（27年度）
3. ID連携まで盛り込み（27年度）

**ブロッカー：**
- マイグレーション過渡期のデータモデル・ポートフォリオ運用が未確定のため、mTG側の仕様が凍結できず開発着手が困難
- Krakenのポートフォリオ運用・データモデル未決定がmTGの会員登録設計に影響
- EA構想との整合性が取れないと直連携で作った機能の手戻りリスク
- mTG直連携がボトルネックとなり新料金リリースが27年度まで長引く（中村社長指摘 Q4ステコミ）

**担当チーム：** inet（BE）、Naikist（画面・BFF）、Guardians（マイクロサービス）
**定例：** 隔週のmTG⇔Kraken定例

## エージェント時代への示唆

AIエージェントの普及により、mTGの利用パターンが変わるリスクがある。

| ステージ | 時期 | 変化 |
| --- | --- | --- |
| Stage 1 | 現在 | ユーザーがガス・電気について、アプリではなくAIに聞く |
| Stage 2 | 2〜3年後 | AIが予約・プラン変更・支払変更などの手続きを代行 |
| Stage 3 | 5年後 | AIが自律的にコスト最適化・タイミング判断を行う |

**mTGの固有データ資産**（使用量データ、機器状態、メンテナンス記録）は東京ガス固有であり、Google/Amazonが持たない。ただし画面経由のみのアクセスではエージェント時代に対応できず、**API-first への転換**が必要。

> 詳細: `assets/context/agent-era-landscape.md`
