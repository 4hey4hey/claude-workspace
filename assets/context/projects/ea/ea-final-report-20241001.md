## 1. サマリー

### (1) EA構想の具現化を通じて実現すること

**契約業務システムの集約：**
- Kraken移行を基軸に、**27年度までにBtoC領域の"契約業務とシステム"を標準化**
- ライフバル、Web販売、BTM、ハウスクリーニング等で培ったノウハウを凝縮して27年度までに**"フィールド業務とシステム"を標準化**
- ソリューション・ロジスティックやAI活用を組み込み、アライアンスパートナー向けに**"競争力のあるクラウドサービス"として販売**
- BtoCの約100システムを30年度に**70システムに削減**、基幹システムとなる契約・受付領域では**50弱を半減**
- カンパニー全体のシステム規模は半減し、**IT人材の余力を創出**

**マーケティング領域：**
- 標準化で集約されたお客さま関連データを活用
- デジタル・リアルが連動した**"重ね売り戦術シナリオのPDCA"を高速化**
- 業務縦割りではない顧客体験を目指し、**会員IDの共通化や会員サイトの刷新**

**最終目標：**
- デジタル・フィールドパートナー・エネルギーコンシェルジュの**三方位から高品質な顧客体験を推進**
- 30年度までにホストコンピュータや自社データセンターからの離脱
- **年間40億円規模のインフラコストを大幅にコストダウン**

### (2) EA推進に求められる役割と体制案

- 経営直下で方針決定を行い、各部長と方針や計画の合意形成を図る
- EA構想を常に最新化した上で、各PJに構想の浸透を図りながら定期的なチェックを実施
- 各種政策とEA構想のバランスをとりながら、各PJに人材・予算等のリソースを最適配置
- カンパニー内のみならず**iNETも含めて一体となって運営できるスキーム**を指向

### (3) 24年度下期の取り組み

- 実現に向けたプロジェクトを新設すべく、目標設定や実現優先度等の構想化と体制整備
- EA推進の役割をより具体化し、推進体制を決定

## 2. EA構想の3本柱

| 柱 | 現状課題 | 実現事項 |
|---|---|---|
| **Krakenを中心とした基幹業務システムの集約** | 商材×業務ごとにシステムが林立し連携が複雑化。業務組織の集約が困難でノウハウも属人化 | システム集約によるデータ一元化と業務・システムの標準化。商材横断の顧客IDで顧客・契約情報を集約管理 |
| **顧客体験・マーケティング強化による重ね売りの推進** | データが各所に点在しリアル・デジタルの顧客接点がバラバラ。マーケ用データが未整備でPDCA高速化が困難。会員サイト・専門窓口が乱立し顧客体験を毀損 | 共通の顧客ID・周辺データの利活用によるリアルとデジタルが融合した顧客体験。顧客理解の深化と1to1コミュニケーション |
| **パートナーへの提供価値拡大による重ね売りの推進** | TGグループ外の企業の当社システム利用のハードルが高い（専用端末の貸与・閉域網の利用が前提） | 各商材のビジネスノウハウを凝縮し、クラウドシステムをパートナーへ提供。任意のデバイス・ネットワークでクラウドサービスとして利用 |

### Krakenを中心とした契約システムとデータ

- エネルギー契約（従量制）
- 定期払い契約（サブスク）
- 都度払い契約（スポット購入）
- 契約・請求管理、顧客ID、契約情報を集約

### パートナー・プラットフォーム

- パートナー向けゲートウェイ
- 受付・余力管理、見積、施工管理、精算
- ノウハウを凝縮（ライフバル販売、Web販売、ハウクリ、BTM、ソリューション・ロジスティクス）

## 4. EA推進に求められる役割と体制

### 他社事例による推進体制の要諦

| パターン | 事例 | 特徴 |
|---|---|---|
| 業務部門一体型 | 素材A社 | ビジネスは部門、基盤は情シス。部門・ITで人材ローテ |
| **社内CoE組織型** | 石油元売りB社 | CDO直下にDX推進組織。業務/IT出身のエース級を集約。DX推進組織が予算確保・執行 |
| 戦略事業子会社型 | 通信C社 | 経営直下で推進。DX推進組織がヒトの配置・育成・採用を担当 |

### 組織設計の要諦と現在のGap

| 要件 | 要諦 | 現在のGap |
|---|---|---|
| **モノ①：迅速な意思決定** | 経営アジェンダに基づいた迅速な優先順位・注力領域の判断 | — |
| **モノ②：構想の浸透** | DX推進組織が構想とPJ推進の両面を担当し個別最適化を抑止 | 全体最適の構想・計画が不在。各部長の考え方に依存 |
| **ヒト：機動的な人材配置** | ビジネス環境の変化に応じた注力領域への集中投下 | 各部・iNETの個別調整のため、リソース配分はなりゆき |
| **ヒト：人材ローテと育成** | ノウハウの局所集中化を避け、全社的なトランスフォームを推進 | CP内ではノウハウ育成の仕組みも意識も希薄 |

### EA推進組織の役割（案）

| 区分 | 役割 |
|---|---|
| **意思決定** | EA構想および計画の最新化。経営・各部長と合意形成 |
| **構想の浸透** | PJへの参画者がEA構想を浸透。レビュープロセスによる定期的なチェック |
| **人材配置** | iNETと一体化しDX人材を管理・配置（カンパニーの優先度、PJの課題等に基づき全体最適化） |
| **人材育成** | 育成に対する責任もEA推進組織に帰属。全社・各部との人材交流を通じてDX人材化を加速 |
| **予算** | PJ予算の編成・機動的配賦（ヒト・カネをセットで最適配置）。ビジネス効果の創出責任は各部に帰属 |

## 5. 24年度下期の取り組み

### 顧客体験・マーケティング強化（主幹：リビング戦略部）

1. 重ね売りを目指したマーケティングの課題認識および目標、政策優先度の整理
2. 政策優先度を踏まえたシステム整備のロードマップ化（①共通の顧客IDおよびCRMの整理、②シングルサインオンを実現する認証基盤、③マーケティング用データマートの整備、④会員サイト統合とマイクロチャネル化）
3. 次年度以降の計画策定と、システム整備に向けた基本構想および予算化

### パートナーとのアライアンス強化（主幹：設備ソリューション事業部）

1. 検討体制の整備、次年度以降の要員投入の検討
2. "パートナープラットフォーム"整備にむけた基本構想（TG-WISP、プロワン、キントーン、ハウスクリーニング専用システム等）
3. 任意のデバイス・公衆網によるシステム利用（ゼロトラスト）トライアル参画と評価
