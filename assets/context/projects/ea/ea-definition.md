# EA構想：用語定義と全体の考え方

## EA（Enterprise Architecture）とは — 東京ガスにおける定義

東京ガスにおけるEA構想は、一般的なIT用語としてのEAよりも具体的かつ事業密着で使われている。

> **東京ガスにおけるEA構想とは：**
> Krakenを中心とした契約業務システムの集約を基軸に、BtoC領域のシステム・業務・データを標準化し、デジタル・パートナー・コンシェルジュの三方位から高品質な顧客体験を実現する全社横断の変革構想。

**具現化を通じて実現すること（2024年10月 最終報告時点の原文）：**

- Kraken移行を基軸に、27年度までにBtoC領域の"契約業務とシステム"を標準化
- ライフバル、Web販売、BTM、ハウスクリーニング等で培ったノウハウを凝縮して27年度までに"フィールド業務とシステム"を標準化するとともに、ソリューション・ロジスティックやAI活用を組み込み、アライアンスパートナー向けに"競争力のあるクラウドサービス"として販売
- BtoCの約100システムを30年度に70システムに削減、基幹システムとなる契約・受付領域では50弱を半減。カンパニー全体のシステム規模は半減し、IT人材の余力を創出
- マーケティング領域では、標準化で集約されたお客さま関連データを活用し、デジタル・リアルが連動した"重ね売り戦術シナリオのPDCA"を高速化するとともに、業務縦割りではない顧客体験を目指し、会員IDの共通化や会員サイトの刷新を図る
- 30年度までにホストコンピュータや自社データセンターからの離脱を図り、年間40億円規模のインフラコストを大幅にコストダウン

## EA構想の3本柱

```
EA構想
├── 1. Krakenを中心とした基幹業務システムの集約
│     └── 契約・受付領域の標準化、データ一元化
├── 2. 顧客体験・マーケティング強化による重ね売りの推進
│     ├── 共通の顧客ID（TG-ID）
│     ├── 認証基盤刷新（IDCS→Auth0）
│     ├── 顧客関連データ参照基盤（新CRM）
│     ├── CDP / マーケツール（Braze）
│     └── 会員サイト統合
└── 3. パートナーへの提供価値拡大による重ね売りの推進
      └── パートナー・プラットフォーム（クラウドサービス化）
```

## SoE / SoR の区分

| 区分 | 意味 | 東京ガスにおける該当領域 |
|---|---|---|
| **SoR**（System of Record） | 基幹システム。契約管理・料金計算 | Kraken、CIRIUS、電力CIS、暮らしES基盤 |
| **SoE**（System of Engagement） | 顧客接点・マーケティング領域 | myTOKYOGAS、認証基盤、TG-ID、CRM/CDP、会員サイト |

2024年9月にSoR領域のEAが確定（「エネルギーの契約・料金計算は全てKrakenに集約」）。
SoE領域については2024年下期から検討開始し、2026年3月のステコミで**プラットフォーム戦略起点に再編**された。

## EA構想の主要キーワード

| 用語 | 意味 |
|---|---|
| **TG-ID** | mTGIDに代わる全社統一ID。mTG以外での利用を想定して再構築。お客さま自身が管理するID |
| **新CRM** | TG-CRM(Salesforce)の用途混在を解消し、マーケティング特化型に転換した顧客関連データ参照基盤 |
| **CDP** | カスタマー・データ・プラットフォーム（Databricks）。推定データを管理 |
| **Braze** | マーケティング配信ツール |
| **データ連携HUB** | 各マスターシステムからのレプリカ統合を行う想定のデータ連携基盤 |
| **エンタープライズデータハブ** | Krakenを中心とした契約データの集約基盤（UPPビジネス） |
| **パートナー・プラットフォーム** | フィールド業務のノウハウを凝縮し、クラウドサービスとしてパートナーに提供する基盤 |
| **重ね売り** | 既存顧客への商材横断の追加販売。GPS（ガス・電力・ソリューション）の拡販 |
| **GPS** | ガス・電力（Power）・ソリューションの略。東京ガスの主要3商材領域 |
| **ホスト脱却** | THANKS、Atom-TES等のホストコンピュータ（メインフレーム）からの離脱。2030年度目標 |
| **幕張データセンター離脱** | 自社データセンターからの離脱。ホスト脱却と合わせて年間40億円規模のインフラコスト削減 |
| **特区** | EA構想PJにおける意思決定プロセスの高速化施策。技術側をNotion/Miroで先行し、ビジネス側ステコミをビジネス判断に集中させる方式 |
