## これは何

TGG版OKR（東京ガスグループ独自のOKR制度）における**制度設計・評価プロセス・運用指針**をまとめた参照資料。ユーザー（杉崎）はスクラムマスター兼マネージャーとして、部下の目標設定支援・四半期フィードバック・年度末評価の当事者であり、AIが制度の原典を踏まえた助言を行うための土台情報。

OKRの作成論（KRの分解、ムーンショット設計、問いの形の活用等）は `okr-methodology.md` を参照。本ファイルは **「制度としてのOKRをどう運用し、どう評価に反映するか」** に焦点を絞る。

## 3行サマリー

1. TGG版OKRは「挑戦による成長」を後押しする仕組みであり、**挑戦KR（変革度×組織貢献度／S〜D）** と **基盤KR（難易度×達成度／5点満点）** の2軸で評価する
2. 期末の業績査定は **「絶対評価→相対評価」** の2段階。**基盤KRを評価ベースに置き、挑戦KRで加点調整**、行動評価と専門性発揮を補助要素とする。上位職ほど成果重視、下位職ほどプロセス考慮
3. 相対評価は部門内で **A:B・C:D以下 ≒ 2:6:2** のガイド値。運用の肝は「前年踏襲」「安住意識」「中心化」「厳しさ不足」という4つの罠を回避すること

## 1. 導入の経緯と目的

### なぜ26年度から基幹職以下に展開されたか

- **期待役割の変化**: グループ員一人ひとりに「中長期を見据えた挑戦を通じて、変革をリードできる人材へと成長すること（＝挑戦による成長）」を期待
- **ミッションシートの4つの課題**:
  1. 小さな目標設定に留まり、挑戦が促進されない
  2. 進捗確認・軌道修正がタイムリーになされない
  3. 難易度・変革度の高い目標へのチャレンジを十分に評価できない
  4. 目標設定で他者を参考にできない（公開範囲が上長のみ）
- **24年度に役員・幹部職に先行導入**し、25年5月時点の効果実感調査で「大きな目標が掲げやすくなった（ストレッチ効果）」が70%と最大
- **26年度から基幹職以下に拡大**した背景：次期中計の実現度を高めるため、上位目標（経営・事業戦略）と個人目標の連動を強化する必要

### 「挑戦による成長」を体現するグループ員像

- 新規領域の開拓や既存分野の変革を実現できる高い専門性、その獲得への強い意欲
- 多様な人材の力を結集して変革をリードできる
- グループ経営理念の価値観「挑み続ける」「やり抜く」を体現し、周囲に好影響を与え続ける

### 従来からの変化要素（MBO/ミッションシート → TGG版OKR）

| 項目 | ミッションシート（〜25年度） | TGG版OKR（26年度〜） |
|---|---|---|
| 目標設定 | 4月一度設定、達成見込みレベルで留まる | 上長OKRと連動、複数年度にまたがるチャレンジング目標含む |
| 進捗確認 | 上期終了時点 + 年度末 | 四半期ごと + 2週間〜1ヶ月の1on1 |
| 評価軸 | 達成度・貢献度の2軸（各5段階） | 難易度×達成度＋変革度×組織貢献度の複合軸 |
| 公開範囲 | 上長のみ（部下目標閲覧可） | 社内全体に公開（評価結果は除く） |

## 2. TGG版OKRの基本構造

### Objective（目標）

- **2030年をターゲットにしたストレッチ目標**（事業ごとの固有事情に応じて最長10年程度まで可）
- 必須要素:
  - **ムーンショット要素**: 実現困難だが達成すれば大きな成果が期待できる挑戦
  - **長期の事業戦略につながる**: Compass2030・中期経営計画との接続
  - **メンバーの行動を促す**: 実現自信度60〜70%、挑戦意欲を引き出す

### 挑戦KR（Challenge KR）

- **複数年度にまたがる変革を伴う通過指標（チャレンジ指標）**
- 必須要素:
  - Objective実現に必要な**変革**を含む（従来のやり方をドラスティックに変えることで実現）
  - 中長期で目指すべき高い成果やチャレンジングな取り組み（新規事業等、成功確率が低くても挑戦価値のあるもの）
- **定性指標も可**（「××制度を改定する」など、Objective達成のプロセスとして明確であれば採用）
- 中間マイルストーンを必ず織り込む
- **最低1つ設定必須**（上長と同一内容でも可）

### 基盤KR（Foundation KR）

- **組織として年度内で確実に達成すべき通過指標（コミットメント指標）**
- 従来のミッションシートの業務遂行目標（個人目標・部下育成）に近いが、**期中進捗を確認できる指標／マイルストーン設定が必須**
- 中計・業計のKPI達成に向けた具体的な取り組みを含む
- **最低1つ設定必須**
- **部下を持つ人は「部下育成」を必須設定**

### 上長OKRとの連動構造

```
[役員・幹部職] Objective（業界トップクラス）
   └ 挑戦KR ─────┐
   └ 基盤KR       │
                  ↓ 実行可能な計画への落とし込み
[基幹職・統括職] Objective（社内トップクラス）
   └ 挑戦KR ─────┐
   └ 基盤KR       │
                  ↓
[指導職・担当職] Objective
   └ 挑戦KR
   └ 基盤KR
```

**重要**: 上位職の目標が十分ストレッチされていないと下位職の目標も小さくなる。上位職ほど業界・社内トップレベルを意識する。

## 3. 目標設定の共通ルール・留意点

### 全社共通ルール

- [x] **全社員が挑戦KRを設定することを目指す**（役員・幹部職OKRの実現性を高めるため）
- [x] **Objectiveや挑戦KRは上長と同一でも可**（変革度担保を優先）
- [x] **部下を持つ人は「部下育成」を基盤KRに必須で設定**
- [x] 突発業務・付加業務・育休者バックアップ等は、必要に応じて基盤KRに追加可
- [x] 難易度「中」以上／変革度「中」以上を期初必須（達成時に基盤KR=4点／挑戦KR=A以上となる水準）

### 部門独自ルール（各部門の裁量）

- 例：部・チームで共通するOや挑戦KRを作成し、メンバー共通化する
- 評価において重視するポイントは部門ごとに差が出るため、不明点は人事担当に確認

### NG例・OK例

| 判定 | 内容 | 理由 |
|---|---|---|
| ❌ NG | 「自信度60〜70%」を理由に当初目標を0.7で割る（100→140を掲げる等） | 実現性が乏しく、数字遊びになる |
| ❌ NG | 測定不可能・抽象的な表現（「検討する」「向上する」「最大化する」等のみ） | 達成状況が共有不可 |
| ✅ OK | 中長期ゴール＋短期マイルストーン併記（例：①3年以内にM&Aで利益倍増／②次年度までに4件検討・2件のPMI完了） | スピード感ある推進と小さな目標化の両立 |
| ✅ OK | 期日・数値の明示 + 前任・前年との比較根拠 | 期末評価の納得感が高い |

### 目標の具体化につながる視点（基準の参考）

| 軸 | 例 |
|---|---|
| 実績 | 売上高、販売台数、市場シェア率 |
| コスト | 人件費、委託作業費、修繕費 |
| 品質・納期 | クレーム件数、遅延率、歩留まり |
| 改善・効率化 | 業務プロセス見直し（〇%削減） |
| 創造性 | 新製品・新システム開発、新市場開拓 |
| 安全 | 災害・事故件数 |

### 目標の具体性を落とす要注意ワード

- 「効率化する／向上する／企画する／推進する／支援する」 → 内容・時期を明確化
- 「図る／管理する／共有化する／定着させる」 → 「どういう状態になればよいのか」を明確化

### 推奨される行動 vs 推奨されない行動（運用の芯）

OKR制度で期待される効果を損なわないための行動指針。部下のOKRをレビューする時、自分のOKRをチェックする時のものさし。

| 観点 | ✅ 推奨される行動 | ❌ 推奨されない行動（避けるべき） |
|---|---|---|
| **テーマ** | 顧客や自分たちが本当に必要・重要・取り組みたいと思えるものをOKRにする | — |
| **上位方針との連動** | 上位方針・上長のKRと自身のObjectiveを連動（トップダウン）させつつ、そこからかけ離れていない表現を意識 | 上位方針を無視し、自身の思いのみに任せたOKRを設定（もしそれだけの熱意があれば上位者に提案しよう）／上位方針に完全に追従し、思考停止したOKRを設定 |
| **難易度** | 全力で取り組めば60〜70%程度の確率で達成できそうな水準のものをOKRとして設定 | 確実に達成できる、あるいは確実に達成できないものをOKRにする |
| **達成に向けたスタンス** | 工夫・諦めずに取り組む姿勢を取り入れる | 旧来通りのリソース・方法に固執し、工夫せず、諦める |
| **期中の変更** | 上位方針・上長のOKRに連動したものとする（一部のみ連動でも可） | 状況変化にかかわらず一切変えない |
| **KRの記載** | Objectiveを測定出来る指標とする | 測定不可能・抽象的な表現で記載する（例：検討する・向上する・最大化する…のみ） |

## 4. 評価軸のメカニクス

### 基盤KR評価マトリクス（5点満点絶対評価）

```
          達成度
         未達  ほぼ達成  達成
難易度 高  3     4        5
       中  2     3        4
       低  1     2        3
```

- 期初は**難易度「中」以上で設定**（達成すると4点以上）
- 期中で早期達成した場合、追加・修正を検討
- **難易度の目安**: 前任や前年実績と比較し、同じレベルに留まる場合は「低」
  - ただし前任・前年より少ないリソース（ヒト・モノ・カネ）で同実績を出す場合は例外

### 挑戦KR評価マトリクス（S〜Dの5段階絶対評価）

```
              組織貢献度
             不十分  貢献  大きな貢献
変革度 大     B      A      S
       中     C      B      A
       小     D      C      B
```

- 期初は**変革度「中」以上で設定**（大きな貢献実現でA以上）
- **組織貢献度は未達でも組織への貢献の大きさで評価**（達成度とは別物）
- 複数年度にまたがる取り組みは適切な中間指標（マイルストーン）で年度内を評価

#### 変革の大きさを測る視点

- 「OやKRの大きさ」「革新性」「新規性」「伸長度」「挑戦姿勢」「執着心」

#### 組織貢献度の大きさを測る視点

- **定量面**: 利益金額・売上金額・コストダウン金額・受注（見込み）件数等の数字
- **定性面**: 仕組み・制度化、新規立ち上げ、業務改善等の内容

### 行動評価（5段階）

- KRを通じた業績面の評価だけでなく、行動・KR以外の組織貢献を総合評価
- 「コミュニケーション」「コンプライアンス」などKRでは測りづらい項目に注目
- 上長による絶対評価

| 評価 | 定義 |
|---|---|
| S | 特筆すべき行動 |
| A | 期待を上回る行動 |
| B | 期待通り |
| C | やや期待を下回る |
| D | 行動改善を要する |

**評価項目（4つの経営理念軸 × 3項目 = 12項目）**:
- 挑み続ける：挑戦する姿勢／競争力の強化／専門性を伸長する姿勢
- やり抜く：業務完遂力／課題形成・企画力／意思決定
- 尊重する：コミュニケーション／指導・育成／公平・公正な姿勢
- 誠意をもつ：将来志向／外部志向／コンプライアンス

## 5. 期中フィードバック（四半期サイクル）

### 年間フロー

```
4月          4-7月          8-10月         11-1月         2-3月
┌────────┐  ┌──────────┐  ┌──────────┐  ┌──────────┐  ┌──────────┐
│目標設定│→ │OKR推進   │→ │OKR推進   │→ │OKR推進   │→ │年度末評価│
│OKR設定 │  │進捗確認  │  │進捗確認  │  │進捗確認  │  │絶対評価  │
│上長承認│  │FB実施    │  │FB実施    │  │FB実施    │  │相対評価  │
└────────┘  └──────────┘  └──────────┘  └──────────┘  └──────────┘
   CIRCLE    CIRCLE ダイナミックノート               STAGE+CIRCLE
```

### 2種類のミーティング

| 種類 | 頻度 | 目的 |
|---|---|---|
| **1on1** | 2週間〜1ヶ月ごと | KR推進上の悩み・進め方の相談、気軽なアドバイス |
| **四半期OKRミーティング** | 四半期末 | KR推進上の課題共有、進捗確認、評価要素を交えた意見交換 |

### 上長のサポート4役割（OKRマネジメントの哲学）

上長は部下のOKRが正しい方向に進むよう後押しし、達成に向けて必要なサポートを行う。4つの役割を場面に応じて使い分ける。

| 役割 | 状況 | 上長の行動 |
|---|---|---|
| **① ガイド**（正しい場所/方向に向かわせる） | 部下が計画から外れている／上位方針が変わった | 随時方向修正（後から理由を問い詰めるのではなく、途中で支援）。上位方針変更はタイムリーにOKRに反映 |
| **② 支援**（部下のみでは進められない領域を支援） | 大きな挑戦、リソース不足 | 財務的資源・人的資源等、部下では解決できない領域に積極的に関与 |
| **③ 助言**（部下が自立して進められるよう） | 部下が自立して進んでいる | OKR推進に向けたフィードバックで気づき・自律を促す／基盤KR遂行に集中しすぎて挑戦KRがおざなりになる等、必要な取り組みの停滞を防ぐ |
| **④ 称賛**（成果や努力の過程を評価） | 進捗や成果が出た時 | KR達成に向けて進捗があった際に、結果だけでなく努力の過程を称賛する |

**重要**: 4つとも「指示」ではなく「サポート」。部下が主役、上長は伴走者。

### OKRミーティングの5W1Hフレーム（今後について話し合う）

現状のギャップを認識した後、次のアクションを具体化する議論で使う。

| 観点 | 問いかけ | ポイント |
|---|---|---|
| **誰が** | 誰がそのタスクを実行するか？ | 主体は部下だけではない。**上長が支援を申し出ることで共通の目標となり、行動に繋がる** |
| **なにを** | すべきタスクは何か？ | 具体的に話し合う |
| **いつまでに** | 期限はいつか？ | 期限設定は確実な遂行のために重要。**短めに設定することでストレッチにもつながる** |
| **どれぐらい** | どの水準か？ | 行動目標であっても、測定可能にすることで行動促進に繋がる |
| **どうやって** | どのように実行するか？ | **なるべく部下に考えさせることで、コミットメントが高まる** |

### 現状の確認と良し悪しを具体的に伝える（ギャップフィードバック構造）

```
  あるべき姿
  ・OKRを実現する為に
  ・更に高い成果を出すために必要となるもの
        ↕ ギャップ
  現状
  ・これまで部下はどういう行動をし、どういった成果を出したのか
```

**具体的に「何が十分で、何が不十分だったのか」を話し合う**ことが出発点。

### 四半期ミーティングで確認する6点

1. 進捗状況（できた/できなかった、できなかった背景）
2. 課題・リスクの共有
3. 解決に向けたアイデア・支援要望のすり合わせ
4. 目標の方向性・修正可否の再確認
5. 次期四半期の展望・チャレンジ共有
6. ダイナミックノート（CIRCLE）への反映

### 四半期ごとの評価の渡し方（例）

```
[第2四半期終了時]
  "現時点での進捗では基盤KRは3点相当"
  → 次のアクションを促す

[第3四半期終了時]
  "第2四半期以降の取り組みで一定の進捗 → 基盤KRは4点相当"
  → さらなる奮起を促す

[期末]
  最終的な実績をもとに点数評価をすり合わせる
```

### GROWモデルによるフィードバック

課題・改善点を伝える「ギャップフィードバック」では以下のフレームが参考:

- **G**oal: 中間目標・評価軸の目線合わせ
- **R**eality: 成功を阻む壁の特定
- **O**ptions: 上長支援の明確化
- **W**ill: 実行計画をダイナミックノートに記載

### 職場健康度・自分健康度

- 1 = 問題あり／2 = どちらともいえない／3 = 問題なし
- OKR達成に向けてモチベーション高く取り組めているか、職場メンバーが過負荷でないかを可視化
- 四半期ミーティングで上長と確認・共有
- 公開範囲は本人・上長のみ

### 挑戦KRが早期達成見込みの場合

- 「100%以上達成見込み」の場合、目標を低く抑えている可能性があり**見直しが必要**
- より貢献度の高い内容に変更する

## 6. 絶対評価→相対評価のプロセス

### 全体フロー

```
[期初]          [期中・期末]         [期末：部門内]          [人事部]
目標設定 ───→  絶対評価      ───→  相対評価とりまとめ ───→ 全体調整
OKR + 専門性     上長評価            基盤KRベース+調整       全体バランス
```

### 絶対評価の3要素

| 要素 | 内容 | 役割 |
|---|---|---|
| **OKRの実績** | 基盤KR（成果中心）＋挑戦KR（プロセス含む） | 主要評価材料 |
| **行動評価等** | 5段階。KR以外の組織貢献、日々の行動も含む | 総合判断 |
| **専門性の発揮状況** | 業績向上に資する専門性の度合い | **補助的参考要素** |

### 相対評価の調整ロジック

**基盤KRの評価をベースに、以下の順序で調整**:

```
Step 1: 基盤KRの点数で序列をつける
Step 2: 同点なら挑戦KRの評価が高い者を上位
Step 3: さらに同点なら行動評価が高い者を上位
        ※人物評価項目（尊重する・誠意をもつ）平均3点未満、
          または「コンプライアンス」が3点未満はStep 3適用外
Step 4: 全て同点なら専門性発揮度の高い者を上位
Step 5: 上記外で特筆すべき業績・組織貢献（周囲のお手本等）があれば考慮
```

### 上位職ほど成果重視、下位職ほどプロセス考慮

- **幹部職**: 挑戦KRは成果（Outcome）を重視
- **担当職**: 挑戦KRは期末までの進捗状況（プロセス）も考慮
- 背景: 下位職ほど挑戦KRの成果を短期で刈り取りにくいため、プロセス評価を厚くする

### 相対評価のガイド値

**部門内の成績分布が概ね `A : B・C : D以下 = 2 : 6 : 2` になるよう目安設定**

#### 幹部職の場合の成績区分（参考）

| 期待を上回る | 期待通り | 期待を下回る |
|---|---|---|
| A / B / C | D / E / F（枠外F/G） | G / H / I（or J/K） |
| 標準+/標準/標準− | 標準+/標準/標準− | 標準+/標準/標準− |

### ガイド運用の留意点

- **ガイド値は目安であり、全段階を使うことが目的ではない**
- 適任者がいなければ、むやみに高い成績をつけず、限られた原資を効果的に活用
- メリハリはあくまで**社員のモチベーション向上のための手段**

## 7. 評価で避けるべき4つの罠

これは運用指針の核心。評価者（上長）として自戒すべきパターン。

| 罠 | 具体症状 | 目指す姿 |
|---|---|---|
| **前例踏襲** | 前年成績を起点に査定。「前年通りの頑張り=同成績」という暗黙の了解 | 当年度の実績と貢献で素直に評価 |
| **安住意識** | 達成できる小さな目標を許容し、達成度が高いから評価も高くする | ストレッチしていない目標は評価上も厳しくする |
| **中心化** | 横並びを意識して突出を回避（上位職ほど強い傾向） | 評価すべき人にはしっかり高い成績を付与 |
| **厳しさ不足** | 嫌われるリスクを懸念して事実を伝えない | 成長機会としてフィードバック。OKR導入企業では評価が下がった部下ほど納得感が高まった事例あり |

### 厳しい査定の基本原則

- **前年度と同水準・同レベルの取り組みのみ**の場合は、**前年度より厳しい成績とすることを基本**とする
- 前年度の査定結果を「発射台」として検討しない

### 部下の成長機会として活用

- 厳しい査定結果となりそうな部下には、期中に不足点・改善点をタイムリーにフィードバック
- 行動評価も絶対評価に含むため、フィードバック範囲が広がっている

## 8. よくある質問（FAQ）

### Q1. 挑戦KRの「変革度」は難易度と同じ？

**No**。難易度だけでなく**「ドラスティックにやり方を変えることで初めて実現できる内容かどうか」**が本質。

例：「紙ベースの上申書を電子化」は難易度は高くないが、業務の進め方が大きく変わり中長期的な波及効果があれば**変革度は高い**と考えられる（押印・保管・スペース削減などの副次効果まで含めて評価）。

### Q2. 挑戦KRの「組織貢献度」と基盤KRの「達成度」の違い

- **達成度**: 当初設定した指標に対する到達状況（未達/ほぼ達成/達成）
- **組織貢献度**: **仮に未達でも、組織にどのような貢献をもたらしたか**で評価する

挑戦KRはストレッチ指標であり達成できないケースも想定される。未達でも**次のアクション・次の事業検討につなげられているか**等の定性面にフォーカスする（新規事業が典型例）。

### Q3. 前任や前年実績と同レベルだと、なぜ難易度「低」？

- 企業の成長が限定的になる／個々人の成長や変革が生まれにくい状況
- 経営理念の「挑み続ける」と整合しない
- **例外**: 前任・前年より少ないリソース（ヒト・モノ・カネ）で同レベルの実績を出す場合は必ずしも「低」にならない

### Q4. 期中に難易度は変わる？

- 基本的には少ない（KPIと連動しているケースが多いため）
- ただし、実際に取り組んでみないと難易度が分からないケースもあり、**期中早期に達成できてしまった場合は難易度の調整や基盤KRの追加設定を検討**
- 四半期ごとのOKRミーティングで「適切な難易度レベルが維持されているか」を確認

### Q5. 第1四半期時点で点数評価が困難な場合は？

- 2030年ターゲットの挑戦KRなど、期初早期に進捗が出にくいケースは**無理に点数評価する必要はない**
- 代わりに**変革度に関する認識合わせ、適切な難易度レベルの確認**を優先
- 納得感ある評価のために、点数評価が可能な指標（中間マイルストーン）を継続的に検討

### Q6. 複数のObjective設定は可能？

- 原則1つ（業務の優先順位付けのため）
- 複数事業を推進する立場なら例外的に複数設定可（例：①A事業を業界トップシェア／②B事業のビジネスモデル確立）

### Q7. 複数のKR設定は可能？

- 可能だが、取り組みの推進力が分散しないよう留意
- 複数KRになる場合も、**基盤KRで5点となるよう複数KRの進捗を総合的に勘案**して点数評価

### Q8. 期中異動の場合の評価は？

- 新職場でOKRを設定し直すのが基本（ただし年度末近くの異動は例外）
- 前任OKRを参考に目標達成の流れを意識
- 年度末評価では**異動前のOKRも把握できるよう対応**（エクセル出力保管 or 新職場OKRに追加）
- 総合評価のパターン例:
  1. 異動前上長の評価結果を引き継ぎつつ、異動後OKRの考課結果と合わせて総合評価
  2. （年度終盤異動）新職場では目標設定せず、取り組みを基盤KRの【付加業績】に追加評価

## 10. スクラムマスター・マネージャーとしての活用ヒント

### team-pulseスキルとの連携

- メンバーのOKRを `assets/context/team-pulse/_members/` に整理する際、基盤KRと挑戦KRを分けて記載
- 四半期ごとのOKRミーティング前に、team-pulseで各メンバーのタスク棚卸し → ai-cosで進捗ストーリー化

### 四半期フィードバック面談の準備観点

1. **事実整理**: 基盤KRの進捗（数字ベース）→ team-pulseで棚卸し
2. **ストーリー化**: 挑戦KRの進捗（定性＋マイルストーン）→ ai-cosで整理
3. **ギャップ特定**: 期初の難易度・変革度と現時点のすり合わせ
4. **次期アクション**: GROWモデル（Goal/Reality/Options/Will）で次四半期の計画
5. **職場健康度**: モチベーション・負荷のチェック

### 部下のKR設計でよくある失敗

- **活動（Activity）を書いてしまう**: 「〜を実施する／〜を検討する」で終わる動詞 → 成果（Outcome）で書き直す
  - NG: 「システム導入を検討する」
  - OK: 「第2四半期末までにシステム導入要件定義を完了し、導入承認を取得」
- **計測不能**: 数値・期日がない → 「KRは測定できるか？」を自問
- **基盤KRと挑戦KRの混同**: 挑戦KRのマイルストーンを基盤KRに置かない（挑戦KRは挑戦KR内で完結させる）

### GMへの進捗報告時の粒度

| KR種別 | 報告のスタイル | 理由 |
|---|---|---|
| 基盤KR | **事実ベース**（数字・マイルストーン達成） | コミットメント指標なので進捗そのものが重要 |
| 挑戦KR | **ストーリー＋変革度の変化** | 未達でも組織への貢献・学びが評価対象。「何が見えてきたか」を伝える |

### 相対評価調整で自分のチームの声を通す

- 同役割資格で複数部下を持つ場合、**期中から相対的な差（どちらが上位か）を意識**した評価を心がける
- Step 1〜5の調整ロジックを踏まえて、自分の部下の評価を次ステップで押し上げる材料（挑戦KRの成果・行動評価・専門性・特筆貢献）を準備しておく
- 特に挑戦KRの「大きな貢献」を定量・定性の両面で記述できる状態にしておくと、部門内調整で強い

## 11. 【補足】OKR作成の実務プロセス（実践ガイドブックより）

部下のOKR設計を支援する時、または自分のOKRを検討する時に使う実務的な手順。

### 11.1 Objective作成の3つのポイント

#### ① 発想の起点（どこから考えるか）

1. **自組織の存在意義を意識する** — どのような成果（業績＋ミッション実現活動）を求められているか、自分はどんな成果を出したいか
2. **全社目標等における位置づけを深掘り** — Compass2030・中期経営計画を実現するために自組織で何をするか
3. **KGI/KPIの意味から考える** — すでに設定されている指標を参考に、「そもそもなぜその指標が設定されたのか」を問い直す（Objective作成のヒントに）

既に取り組んでいるテーマを引き続きObjectiveとする場合は「なぜ実現できていないのか」「継続する意味があるのか」を自問し、課題をヒントにする。

#### ② 表現方法（どう書くか）

1. **なるべく簡潔に記載** — 暗唱できるぐらいの文章が理想
2. **ワクワクできるイメージを表現** — 実行するのは社員自身。成功時の姿を記述し、動機づけに繋げる
3. **具体的な期日を記載** — 「3年後に」「20XX年〇月までに」で実効性が高まる

#### ③ 作成の進め方（誰と作るか）

トップダウン（上長KRとの連動）とボトムアップ（所属メンバーの納得）の両方をバランスよく取り入れる。

```
[Step 1] 上長のOKRを担当部門・メンバーに共有
    ↓
[Step 2] 部下・職場メンバーと上長OKRを「読み解く」
         — KRで掲げられた数字に対して、自部所として貢献できる取り組みはあるか
         — その取り組みで、どのような将来像が実現できるか
         — 将来像は中計等の上位方針につながっているか
    ↓
[Step 3] 読み解きを踏まえ、メンバーとともに自組織のObjective案を作成
         — 上長の挑戦KRと関連するストレッチしたObjective案を検討
         — メンバーの思いや「個人が目指したい目標」との共通項を反映
    ↓
[Step 4] 対話を踏まえたOKRの詳細化
         — メンバーと設定したObjective案を自信度等も考慮し微修正
         — 完成したObjectiveをもとに挑戦KR・基盤KR案を作成
```

### 11.2 Objective参考事例（10パターン）

実践ガイドブック掲載の例。自分・部下の期待役割に沿って参考化する。

1. 20XX年度までに**●●の分野で業界シェアNo1**となる
2. ○○分野において**A社を超えるブランド認知度**を獲得する
3. 1年間で**若年層（20代）からの売上を前年度比3倍**まで拡大する
4. 2030年までに▲▲という**課題を解決するソリューション・ビジネスモデル**を開発する
5. 2040年までに**既存の●●事業に代わる新たな事業の柱**を確立する
6. **社員数XXX名で利益をあげられる体制**を3年以内に構築する
7. **同業種内で国内トップレベルの生産性**を達成する
8. 5年以内に**働きがいのある会社として国内大企業の中でNo1**となる
9. ▽▽事業において**国内トップクラスの人材が集まる組織**を作る
10. ベンチマーク企業の事例も参考に、**〇〇業務の進め方を抜本的に変える変革**を実現する

### 11.3 KR作成時のObjective分解（2パターン）

Objectiveを測定可能なKRに落とし込む時の分解アプローチ。

#### パターン① 複数指標をクリアする必要があるケース

Objectiveの達成に**複数の指標クリアが必要**な場合。

- 例1：「**事業の柱を確立**」→ 売上・利益・市場シェア等、複数の指標クリアで初めて「柱」と言える
- 例2：「**高性能な製品を開発**」→ 複数の製品試験（処理スピード、正確性等）でXX点以上で判断

→ ある時点での**「売上・利益・市場シェア」や「製品試験のスコア」がKRの候補**

#### パターン② プロセスを1つずつクリアする必要があるケース

Objective達成に**段階的なステップ**がある場合。

- 例1：「**変革を実現**」→ 構想・計画・要件定義・開発・テストを1つずつクリアする必要
- 例2：「**ブランド認知度の向上**」→ 注意・関心・欲求・記憶・行動（消費）と段階があり、どの段階を課題と捉えるかを明確化

→ ある時点で「**Aというステップをクリアしている**」や「**Bという段階で数値が向上している**」等がKRの候補

**注意**: 分解する中でKRを探す時、「成果の出やすい指標（＝小さな目標）」になりすぎないよう留意。中長期ゴール＋短期マイルストーンを併記する。

### 11.4 挑戦KR設定の思考プロセス（5ステップ）

```
[1] Objective: 「24年度末までに、新製品Xが市場から高い評価を獲得し、
               社会に新しい価値を提供できている」
    ↓
[2] Oに至る変革を実現していく上での通過点は何か？
    "市場からの高い評価"とは？ "新しい価値の提供"とは？
    ↓
[3] ・市場からの評価は導入企業数や売上（=量）で測定できそう
    ・新しい価値の提供は、導入顧客の満足度（=質）で測定できそう
    ↓
[4] どうやって測定する？ 適切な数値目標は？
    ↓
[5] 挑戦KR: 「第1四半期末時点で新製品Xが30社に導入され、
             5段階のユーザーアンケートで、使い勝手の項目が4.0以上」
```

**挑戦KRは山登りの道標**。登山家に例えれば、高い山への登頂を達成するための山登りの道標。

### 11.5 KR検討の自問自答チェック（挑戦KR・基盤KR共通）

KR設定後、以下の問いで自分のKRを検証する。

#### 達成時の評価に関する確認

- [ ] **KRが全て出来たとして「Oが出来た」と胸を張って言えるか？**
- [ ] **KRは「活動」ではなく「成果（測定可能なアウトプット）」になっているか？**
  - 実施／相談／分析／参加…など**動詞で終わる文章は活動である可能性が高い**
  - 不十分な場合は、Oを再度要素に分解して、個別に測定可能なものを検討

#### 測定可能性・難易度の確認

- [ ] **KRは測定出来るか？**（期限や数値が設定されているか）
- [ ] **KRの難易度は適正か？**（難しすぎ、簡単すぎもしない水準か）

数値化や期日設定がない場合、進捗状況が共有しづらくなり、フィードバック時に上長・部下間で認識の相違が生まれる。

### 11.6 ダイナミックノートへの記載のコツ

- **項目名（テーマ）のみKR欄に記載し、詳細は「補足」欄に記載**（補足は本人・上長のみ公開）
- 具体的な件名やお客さま名は補足欄を活用
- 取扱注意情報（レベルA、行為規制該当）はシステム外で共有
