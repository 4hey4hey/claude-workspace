## 1. 何が起きているか：AIブラウザエージェントの一斉投入

2025年後半から主要AIプレイヤーがブラウザエージェント（ユーザーの代わりにWebサイトを操作するAI）を一般提供し始めた。2026年3月時点で「誰でも使える」状態にある。

### 主要プレイヤーの動向（2026年3月時点）

| プレイヤー | サービス名 | リリース時系列 |
|---|---|---|
| **Anthropic** | Claude in Chrome | 2025年8月：研究プレビュー（1,000名）→ 11月：Max全員 → 12月：全有料プラン（Pro・Max・Team・Enterprise）にベータ提供 |
| **OpenAI** | Operator → ChatGPT Agent | 2025年1月：Operator公開（Pro限定）→ 7月：ChatGPT Agentへ進化 → 8月：Operator終了 → 10月：ChatGPT Atlas（専用ブラウザ）公開 |
| **Google** | Gemini in Chrome（auto-browse） | 2025年9月：Gemini in Chromeを米国一般提供 → 2026年1月：auto-browse（自律ブラウジング）プレビュー公開。Universal Commerce Protocol（UCP）をShopify・Etsy等と共同策定 |
| **Microsoft** | Copilot Tasks / Edge Browser Actions | 2025年9月：Edge内Browser Actions導入 → 2026年2月：Copilot Tasks（クラウド型自律エージェント）プレビュー公開。クラウド上で永続実行し、ユーザーのローカルマシンを占有しない |
| **Apple** | Siri（次世代版） | 2026年6月WWDC26で発表予定。Geminiベースの会話型AIエージェント。アプリ操作・Web検索を自律実行する方向。iOS 27で提供見込み |

### 時系列の意味

- **2026年前半（現在）：** PC/ブラウザベースのエージェントが有料プランで利用可能。技術に関心のある層が利用中
- **2026年後半〜2027年：** Apple（Siri）・Google（Gemini）がスマートフォンOSに標準搭載。「意識せず使う」層が急拡大
- **2028年〜：** エージェント間取引・自律購買が本格化（Gartner予測）

## 2. お客さまの行動変化：3段階モデル

お客さまがAIエージェントを使う過程は段階的に進む。一気に切り替わるのではなく、「聞く」→「任せる」→「委ねる」と深化する。

### 第1段階：AIに聞く（情報検索の代理）— 既に発生中

お客さまは東京ガスのサイトを開かず、AIに質問する。

- 「先月のガス代いくらだった？」
- 「東京ガスと〇〇ガスどっちが安い？」
- 「引越し先でガスを使うにはどうすればいい？」

AIはmTGの画面を読み取るか、Web情報を集めて回答。お客さまはアプリを開く必要がなくなる。

**影響：** 情報閲覧目的のアクセス（FAQ・料金案内・使用量確認）が構造的に減少する。

### 第2段階：AIに操作を任せる（単純操作の代理）— 2〜3年以内

お客さまはAIに具体的な操作を指示する。

- 「引越し先のガスの開栓を予約して」
- 「来月から電気も東京ガスにまとめて」
- 「支払い方法をクレジットカードに変更して」

AIがWebサイトやフォームを操作して手続きを完了。お客さまは結果確認のみ。

**影響：** 手続き系の画面遷移が減少。ファネル（到達→閲覧→クリック→申込）の入口が細くなる。

**普及の見通し：** Apple Intelligence・Google Geminiのスマホ統合により、2〜3年以内に一定割合が利用開始。ただし本人確認・認証の仕組みが整わないと普及は限定的。日本の規制環境・高齢顧客層を考慮すると、3年後時点で5〜6割の確率で無視できない規模に到達。

### 第3段階：AIに判断を委ねる（意思決定の代理）— 5年以内（若年層）

お客さまは目的だけ伝え、AIが最適な判断と実行を行う。

- 「うちのエネルギーコストを最適化して」
- 「光熱費を月1万円以下に抑えて」
- 「給湯器が壊れそうなら、良いタイミングで交換手配して」

AIが複数社を比較し、最適なプラン・タイミング・手続きを自律的に選択。お客さまは東京ガスを「選ぶ」のではなく、「AIに選ばれる」立場になる。

**普及の見通し：** 3年後はまだ限定的（15〜20%の確率）。5年後には若年層を中心に家計自動最適化サービスとして定着開始の可能性。

### Gartner

| 発表時期 | 予測内容 | 時間軸 |
|---|---|---|
| 2024年2月 | 検索エンジン経由トラフィックが25%減少 | 2026年末まで |

| 2025年8月 | エンタープライズアプリの40%にタスク特化型AIエージェント搭載（2025年は5%未満） | 2026年末まで |
| 2025年10月（IT Symposium） | **B2B購買の90%がAIエージェントで仲介。15兆ドル超がAIエージェント経由で処理** | 2028年まで |
| 2025年10月 | agentic AIプロジェクトの40%以上が中止される | 2027年末まで |

#### Gartner B2B予測の詳細（2025年10月）

この予測の核心は3点。

1. **「データフィード経済」の出現：** AIエージェント同士が交渉・契約・購買を高頻度で自律実行する世界では、検証可能な運用データそのものが通貨になる。デジタル信頼フレームワーク（データの正当性・鮮度・出所を機械的に検証する仕組み）を持たない企業は、エージェント間取引から排除される。
2. **API-first・ヘッドレスが競争の堀：** AIエージェントはUI（画面）ではなくAPIを通じてサービスにアクセスする。コンポーザブルなマイクロサービス、API-first、クラウドネイティブ、ヘッドレスで設計された製品が構造的な競争優位を確立する。UIしか持たないサービスはエージェントに「選ばれない」リスクを負う。
3. **販売サイクルの根本的圧縮：** AIエージェントが人間の介在なしに交渉・比較・購買を完了するため、従来の営業プロセス（問い合わせ→見積→比較検討→契約）が大幅に短縮される。

**B2C（東京ガス）への含意：** B2B予測だが、B2Cでも同じ構造変化が起きる。AIエージェントが「お客さまの代理人」としてエネルギー会社を比較・選択する世界では、①東京ガスのデータがAPI経由でエージェントに提供できること、②提供データの信頼性・検証可能性が担保されていること、が「選ばれる」ための前提条件になる。

## 5. 東京ガスの構造的ポジション

### 強み：他社が持っていない固有データ

東京ガスが持つ以下のデータは、GoogleもAmazonも持っていない。AIエージェントが最も賢い提案をするにはこのデータが不可欠。

| データ種別 | 内容 | 固有性 |
|---|---|---|
| エネルギー消費データ | 家庭のガス・電気の使用量推移、前年比較、季節変動 | ◎ 東京ガス顧客のみ |
| ガス機器稼働データ | 給湯器・床暖房の稼働状況、利用パターン | ◎ 物理接点あり |
| 点検・メンテナンス情報 | 訪問点検で得た機器の状態、設置年、劣化度 | ◎ 物理点検が必要 |
| 契約・料金データ | 契約種別、料金プラン、切替履歴 | ○ 基幹システムに蓄積 |
| 地域データ | ガス管内の地域特性、世帯構成推定 | ○ 管内800万世帯 |

### リスク：データがAPI化されていない

現時点で上記のデータはmTGの画面上でのみ表示されている。APIとして整備されていない場合：

- AIエージェントは画面をスクレイピングして勝手にデータを読み取る（品質・可用性のコントロール不能）
- または東京ガスを無視して他社のデータだけで判断する（「選ばれない」リスク）

## 6. 「毎日使いたくなるアプリ」戦略の前提変化

### 現在の方針

東京ガスの中経・3ヶ年ロードマップは「月1回の料金確認アプリ」→「毎日使いたくなるプロダクト」への転換を目指している。EA構想は「暮らしの統合プラットフォーム」として接点拡大を描いている。

### エージェント時代に崩れる前提

| 前提 | 崩れる理由 |
|---|---|
| お客さまがアプリを開いて情報を見る | AIに聞けば画面を見なくて済む |
| 毎日開く理由になる機能を追加すればMAUが上がる | ガス・電気は天気やニュースと違い毎日確認する必然性がない。エージェント時代にはさらに成立しにくい |
| アプリの画面上でクロスセル導線を踏ませる | エージェントが代理操作する場合、バナーやCTAは見ない |
| MAUが顧客接点の質を測る指標として有効 | エージェント経由の接点はMAUにカウントされない |

### 検討されている「日常利用」施策とエージェント時代の評価

2023年からユーザーリサーチで検証された9つの価値仮説のうち、需要度が高かった上位3つ：

| 施策 | ユーザー需要 | エージェント時代の評価 |
|---|---|---|
| 外出先からの室温管理 | 最も高い | メーカーアプリ・Apple Home/Google Homeと競合。東京ガスのガス機器データとの連携がなければ差別化不可 |
| 節電ゲーミフィケーション（SEGA協業検討） | 子持ち家庭に好評 | 使用量データを使えば固有価値はあるが、ゲーム自体は東京ガスでなくてもできる。専業との品質競争が発生 |
| 家事の見える化・管理 | 社会課題として重要 | 東京ガスの固有データとの接点がゼロ。TimeTree・Yieto等の専業アプリが存在。ガス会社への期待と不一致 |

## 7. 効果が逓減する施策パターン（レビュー時の参照用）

以下のパターンに該当する施策は、エージェント時代に構造的に投資対効果が悪化する。

| パターン | 代表例 | 逓減の理由 |
|---|---|---|
| **開かせるためのギミック** | ポイント・ガチャ・スタンプラリー | お客さまの生活を良くしていない。エージェント代理利用の世界で画面を開かせる意味がなくなる |
| **汎用コンテンツ** | レシピ・暮らしコラム | 東京ガスでなくても読める。AI検索に吸われる典型（CTR 61%減） |
| **パーソナライズされていない情報** | 天気連動の省エネTips | 一般的すぎる。AIの方が個人に最適化された提案を返せる |
| **大規模経済圏との消耗戦** | クーポン・ECモール連携 | dポイント・楽天・PayPayと還元率で競争する体力がない |
| **既存の強い代替がある領域** | 家計簿、天気予報、防災情報 | マネーフォワード・Yahoo!天気等が圧倒的に強い。mTGに移す合理性がない |
| **コミュニティ系** | 掲示板・ユーザーフォーラム | ガス会社アプリでコミュニティが成立する世界線はほぼない。LINE・X・Instagramに確立済み |

## 8. 施策が持つべき要件（評価基準の根拠）

エージェント時代に耐える施策は、以下の特徴を持つ。

### 必須要件

1. **東京ガスの固有データが中核にあること** — 他社が真似できないデータ（エネルギー消費・機器稼働・点検情報）を使って初めて成立する機能であること
2. **チャネル・KPI非依存であること** — アプリ画面に閉じず、ロジック・データがAPI経由でもエージェントに提供できる設計であること。KPIもMAU/DAUだけでなく、データ提供同意率・NRR・チャネル問わずの転換率で測ること。今作るロジックが将来エージェント向けに転用できる設計であること

### 推奨要件

3. **競争のモートを築くこと** — エージェントが比較購買する世界で「東京ガスが選ばれる」理由を作ること

### 認証基盤の文脈

OAuthへの移行（現在Auth0への刷新が進行中）は、エージェント対応の土台になる。基本計画の中で、エージェントへの権限委任を将来追加できるスコープ設計を検討事項に含めることが望ましい。追加コストはほぼゼロ。

## 9. 補助指標の提言

MAUに加え、以下を補助指標として追加検討すべき。

| 指標 | 意味 | なぜ必要か |
|---|---|---|
| **データ提供同意率** | お客さまが東京ガスのデータをAIエージェント等に提供することに同意した割合 | エージェント時代の「接点の質」を測る |
| **既存顧客売上維持率（NRR）** | 既存顧客からの売上がどれだけ維持・拡大されているか | エージェント経由のSW・アップセルも含めた収益指標 |
| **チャネル問わずの転換率** | 特定チャネル（画面）に依存しない、全経路での契約転換率 | 画面ファネルが細くなっても、全体の効果を見失わないため |
