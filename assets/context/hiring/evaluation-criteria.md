# 面接評価基準

> **対象ポジション:** デリバリーマネージャー
> **最終更新:** 2026-04-12
> **Mission接続:** 「作る力を、届く価値に変える」—— DMはSMのシニア版＋ピープルマネジメント。SMがチーム内で回すことを、横断で標準化・型化し、SM自身の成長も支援する。行動原則「守る・速くする・強くする」に直結した5軸

---

## スコアリング方式

各軸を1〜5で評価し、根拠となる発言/事実をメモする。

| スコア | 定義 | 行動レベル |
|--------|------|----------|
| 1 | 不足 | 概念のみ。具体的な実践がない。再現性なし |
| 2 | 経験あり | 1チームで経験あるが属人的。意図的・体系的でない |
| 3 | 一定の再現性 | 1チームで意図的に回し、型/運用を残した |
| 4 | 高い再現性 | 複数チーム/プロジェクトで回し、型として定着させた |
| 5 | 強い再現性 | 組織レベルで型を作り、自分がいなくても回る状態を残した |

---

## 5軸評価基準

### A. 観測力（守るの前提）

**定義:** 複数チームを横断的に観測し、崩れる前の兆候を捕らえられるか。フロー指標（リードタイム/滞留/手戻り率）と人の兆候（発言偏り/抱え込み/トゲ）の両方を、チーム横断で見ているか。

**検証するMust:** M3, M4

| Lv | 行動 |
|----|------|
| 1 | 「なんとなく遅い」「雰囲気が悪い」という感覚レベルのみ |
| 2 | 遅延やトラブル発生後にデータを見る（事後的） |
| 3 | リードタイム/サイクルタイム/WIP等の定量指標を定点観測している |
| 4 | 定量指標＋行動面（発言偏り、抱え込み、レビューのトゲ）の両面を見ている |
| 5 | 定量＋定性の観測体系を作り、チームやリーダーにも観測の習慣を移植している |

**対応する面接質問:** セクション2-1（ムダの観測と除去）S/T、セクション2-2（健全性モニタリング）S/T

**危険信号:**
- 数字を一切出さない（「体感で」「肌感覚で」が多い）
- 問題が顕在化してからしか動かない
- 数字しか見ない、または空気しか見ない（片方に偏る）

---

### B. デリバリー設計力（守る：ムダ）

**定義:** 横断視点で流れを整えられるか。チーム横断でムダ（手戻り・使われない機能）を観測しSM/POにFB。フロー改善を推進し、SMの打ち手をレビュー。「小さく分割して学ぶ」前提で設計できるか。

**検証するMust:** **M5**

| Lv | 行動 |
|----|------|
| 1 | ビジネス側の要求をそのまま開発に流す「調整役」 |
| 2 | 問題は認識するが、打ち手が「頑張る」「意識する」で終わる |
| 3 | ボトルネックを特定し、Ready条件/WIP制限/変更管理など運用に落とした改善を実施。SMの打ち手に対してFBできる |
| 4 | ビジネス側に対して「小さく分割して優先順位をつける」と言える。PO/経営/他部門への働きかけを含む介入ができる |
| 5 | ムダを流さない仕組みを型化し、チームが自走で入口品質を管理できる状態を作った |

**対応する面接質問:** セクション2-1（ムダの観測と除去）全体、セクション2-3（仮説検証サイクル）、ケースK5（調整役に堕ちないか）

**危険信号:**
- ビジネス側の要求をそのまま受け入れる（「全部やります」）
- 打ち手が「ミーティングを増やした」「共有を強化した」のみ
- 仮説検証型でなくQCD管理型のアジャイル経験のみ
- 「自分がやった方が早い」で介入を完結させる

---

### C. 型化・標準化力（強くする：ムラ）

**定義:** SMがチーム内で回していることを、横断で標準化・型化できるか。属人を剥がし複数チームで回る型を残せるか。改善を1回で終わらせず、効果検証→定着→次の詰まりまで見続けられるか。

**検証するMust:** M3

| Lv | 行動 |
|----|------|
| 1 | すべて自分が直接対応する。「自分がいないと回らない」 |
| 2 | 対応した内容を文書化はするが、他者が使える形にはなっていない |
| 3 | テンプレ・チェックリスト・運用ルールとして型を残している（1チーム内） |
| 4 | 型を別チームに横展開し、展開先でも機能することを確認。ボトルネック移動を前提に次の観測点を設計 |
| 5 | 型の展開プロセス自体を標準化し、増員/新チーム立ち上げで再利用されている |

**対応する面接質問:** セクション2-3（チーム立ち上げ・型化）全体、セクション2-4 R（結果）、セクション2-1 R（結果）

**危険信号:**
- 「メンバーが優秀だったのでうまくいった」
- 型が個人依存で、横展開の視点がない
- 「まず走りながら考えた」で基盤がないまま終わる
- 1回の施策を武勇伝として繰り返し語る（効果検証/継続モニタリングがない）

---

### D. ピープル力（守る：ムリ ＋ 強くする）

**定義:** 人を守り、育てられるか。持続可能性の守護（過負荷防止・バーンアウト予防）、リーダー・SMの育成（答えを渡すのでなく行動変容を支援）、コンディション管理。

**検証するMust:** **M4**

| Lv | 行動 |
|----|------|
| 1 | 「頑張ってもらった」「アドバイスした」で終わる |
| 2 | 1on1はやっているが、観測・介入の設計がない |
| 3 | 課題検知→施策→結果のサイクルがある。成果を一時的に落としてでも守る判断ができる |
| 4 | リーダー・SMの行動変容を支援（同席・テンプレ・場の設計・権限委譲）し、変化を観測している |
| 5 | 育成の型を作り、自分が抵けてもリーダー・SMが次の人を育てる状態を作った |

**対応する面接質問:** セクション2-4（リーダー支援）全体、セクション2-2（健全性モニタリング）、業績不振メンバーへの対応

**危険信号:**
- 「本人にもっと頑張ってもらった」
- 「自分が代わりにやった」
- 変化の確認（観測）がない
- 問題を個人の能力・性格に帰結させる

---

### E. 信頼構築力（つなぐの土台）

**定義:** 権限なしで信頼を作れるか。新環境への適応力。対立時に論点と感情を分離し対話で前に進められるか。異なる立場（PO/Dev/ビジネス）をつなげるか。

**検証するMust:** 共通要件

| Lv | 行動 |
|----|------|
| 1 | 対立を権限で押す or 回避する。自分の非を認められない |
| 2 | 表面的には協調的だが、深い対立場面での対処が語れない |
| 3 | 対立場面で傾聴→構造化→合意形成の型がある。異論を潰さない |
| 4 | 権限がない状態でも信頼を築き、影響力を発揮した具体事例がある |
| 5 | 対話姿勢自体をチームに浸透させ、心理的安全性の高いチームを作った実績がある |

**対応する面接質問:** セクション1（適応・協働）全体、対話姿勢セクション、対立対応STAR全体

**危険信号:**
- 相手の態度を直接指摘する（「感情的にならずに」）
- 正論/ロジックで説得・論破しようとする
- 違和感を建設的に捕らえられない（批判止まり）
- 権限がないと動けないことを示唆する発言
- 過去の成功パターンを文脈分離せず適用しようとする（NG-E）

## 総合判定基準

| 判定 | 条件 |
|------|------|
| **採用** | A〜Eすべて3以上、かつB（デリバリー設計力）とD（ピープル力）のいずれかが4以上 |
| **保留** | BまたはDが2以下だが、他で補える可能性あり。次回面接で重点確認 |
| **見送り** | B（デリバリー設計力）が2以下、またはD（ピープル力）が2以下 |

**補足判定ルール:**
- A〜Eのいずれかが1の場合 → その軸の理由を明示し「保留」以下
- Eが2以下 → 他軸が高くても「見送り」候補（権限なしで動けないDMは配属後に機能しない）
- 危険信号が3つ以上 → 合計スコアに関わらず「見送り」
