# issue Eval基準

## 評価軸（3軸）

### 軸1: 多視点の網羅性 — Perspective Coverage Score

4視点（経営・開発・顧客・ガバナンス）が実質的にカバーされているか。

| スコア | 基準 |
|---|---|
| **3** | 4視点すべてが具体的かつ固有の指摘を含む。情報限定的な視点は「⚠️情報限定的」と明記。 |
| **2** | 4視点出力されているが、1-2視点が表面的（一般論の繰り返し）。 |
| **1** | 2-3視点のみ。一部の視点が欠落。 |
| **0** | 多視点レビューが実行されていない。 |

**合格ライン: 2以上**

### 軸2: クロスカットの統合度 — Cross-Cut Quality Score

Phase 3（クロスカット統合）が視点間の関係を実際に分析しているか。

| スコア | 基準 |
|---|---|
| **3** | 視点間のトレードオフが明示され、合意点と懸念点が整理されている。総合判定に根拠がある。 |
| **2** | トレードオフの列挙はあるが、優先順位や判断基準が不明確。 |
| **1** | 4視点の結果を並べただけ。横断分析なし。 |
| **0** | クロスカットが実行されていない。 |

**合格ライン: 2以上**

### 軸3: 抽出の深さ（Phase 1）— Extraction Depth Score

課題構造化モード: ユーザーのモヤモヤから課題の輪郭を引き出せたか。
ドキュメントレビューモード: ドキュメントの主張・前提・暗黙の仮定を抽出できたか。

| スコア | 基準 |
|---|---|
| **3** | 明示された情報だけでなく、暗黙の前提・言外の本音も引き出している。問い返しが的確。 |
| **2** | 明示情報の構造化はできているが、暗黙の前提は見逃している。 |
| **1** | ユーザーの入力をそのまま繰り返し。深掘りなし。 |
| **0** | Phase 1をスキップしてPhase 2に進んでいる。 |

**合格ライン: 2以上**

## 総合

| モード | 評価軸 | 合格基準 |
|---|---|---|
| 課題構造化 | 軸1 + 軸2 + 軸3 | 全て ≥ 2 |
| ドキュメントレビュー | 軸1 + 軸2 + 軸3 | 全て ≥ 2 |
