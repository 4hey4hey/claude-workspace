# ppt Eval基準

## 概要

ppt統合スキル（設計→承認→生成）の出力品質を測定するための評価基準。
Skill Optimizer が改善提案の効果を検証する際に使用する。

---

## 評価軸（2軸）

### 軸1: 設計テンプレ完全性 — Template Completeness Score

Phase 1 で生成される設計テンプレートが、Phase 2 の生成に必要な情報を全て含んでいるかを評価する。

| スコア | 基準 | 判定ポイント |
|---|---|---|
| **3** | ゴール・ストーリーライン・全スライドのassertion・「だから何？」・ボディ要素・図解タイプ指定が完備。pptx-creatorが追加判断なしに生成可能。 | テンプレだけ読めばデッキの全貌が分かる |
| **2** | 主要フィールドは埋まっているが、一部のスライドで「だから何？」やボディ要素が曖昧。 | pptx-creatorが「推測して補完する」箇所が1-2件 |
| **1** | assertionがカテゴリラベル、またはボディ要素の大半が未指定。 | テンプレがスライドタイトル一覧に留まっている |
| **0** | 設計テンプレなしでPhase 2に進行。 | — |

**合格ライン: 2以上**

**チェック方法:**
1. ゴール文（「誰が何を判断する」）が明記されているか
2. 全スライドにassertion（主張文）があるか（ラベルではない）
3. 全スライドに「だから何？」があるか
4. ボディ要素（テーブル/カード/フロー/比較等）が指定されているか
5. 図解が必要なスライドに `diagram:タイプ` が指定されているか

---

### 軸2: Phase承認ゲート遵守 — Phase Gate Compliance Score

Phase 1→Phase 2の移行時に必ずユーザー承認を得ているかを評価する。

| スコア | 基準 | 判定ポイント |
|---|---|---|
| **3** | Phase 1 完了後にテンプレを提示し、AskUserQuestionで承認を得てからPhase 2に進行。Phase 2中にテンプレ変更が発生した場合も再承認を取得。 | 承認なしの自動進行が一切ない |
| **2** | Phase 1→2の承認は取得しているが、AskUserQuestion未使用（テキストで確認）。 | 形式は不完全だが停止はした |
| **1** | Phase 1 の一部ステップ（Step 2→3等）で確認をスキップ。 | テンプレ不完全なままPhase 2に進んだ |
| **0** | Phase 1→2を一括出力。承認ゲートなし。 | — |

**合格ライン: 3（承認ゲートは必須要件）**

**チェック方法:**
1. Step 2 後（タイトル一覧）に確認があるか
2. Step 3 後（ボディ設計完了）にテンプレ全体の承認があるか
3. Phase 2a（図解）の着手前にdraw.ioプレビュー承認があるか
4. Phase 2b（PPTX生成）の着手前にテンプレ最終承認があるか

---

## 総合スコア

| 評価軸 | 合格基準 |
|---|---|
| 軸1 + 軸2 | 軸1 ≥ 2 かつ 軸2 = 3 |
