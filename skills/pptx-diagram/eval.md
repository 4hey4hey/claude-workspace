# pptx-diagram Eval基準

## 概要

pptx-diagramスキル（draw.io→PNG→PPTXスライド埋め込み）の出力品質を測定するための評価基準。
Skill Optimizer が改善提案の効果を検証する際に使用する。

---

## 評価軸（2軸）

### 軸1: プレビュー承認ゲート遵守 — Preview Gate Compliance Score

draw.io XML生成後、PNG変換前にユーザーのプレビュー確認を得ているかを評価する。

| スコア | 基準 | 判定ポイント |
|---|---|---|
| **3** | draw.io XML保存後にプレビュー確認を求め、AskUserQuestionで承認を得てからPNG変換に進行。 | ユーザーがVSCodeプレビューで内容を確認した上で変換 |
| **2** | プレビュー確認は求めたがAskUserQuestion未使用（テキストで確認を求めた）。 | 停止はしたが形式が不完全 |
| **1** | プレビュー確認をスキップし、draw.io XML→PNG を一括実行。 | ユーザーが図の内容を確認する機会がなかった |
| **0** | draw.io XMLの生成自体をスキップ（直接PNG/SVGを埋め込み）。 | — |

**合格ライン: 3（プレビューゲートは必須要件）**

**チェック方法:**
1. `assets/diagrams/{name}.drawio` ファイルが生成されているか
2. PNG変換コマンド実行前にユーザー確認があるか
3. AskUserQuestionツールが使われているか

---

### 軸2: レイアウト精度 — Layout Accuracy Score

PNG画像のPPTXスライドへの埋め込みが、座標ガイドラインに準拠しているかを評価する。

| スコア | 基準 | 判定ポイント |
|---|---|---|
| **3** | アスペクト比が正確に保持。コンテンツ領域（y=1.25〜5.45）内に収まる。ヘッダー/フッターとの重なりなし。色コントラストルール準拠（濃色背景には白文字）。 | 視覚的にプロフェッショナルな仕上がり |
| **2** | アスペクト比は保持されているが、コンテンツ領域からわずかにはみ出し、または色コントラストが一部不適切。 | 微修正で使える |
| **1** | アスペクト比が歪んでいる、またはヘッダー/フッターと明確に重なっている。 | 再生成が必要 |
| **0** | 画像が表示されない、またはスライド外に配置されている。 | — |

**合格ライン: 2以上**

**チェック方法:**
1. addImage の w/h が元画像のアスペクト比を保持しているか
2. y座標 ≥ 1.25、y + h ≤ 5.45 の範囲内か
3. 濃色要素に白文字、薄色要素にダーク文字が使われているか
4. slide.background が `{ color: 'FFFFFF' }` か

---

## 総合スコア

| 評価軸 | 合格基準 |
|---|---|
| 軸1 + 軸2 | 軸1 = 3 かつ 軸2 ≥ 2 |
