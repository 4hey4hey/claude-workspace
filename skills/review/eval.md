# review Eval基準

`review` は `issue` スキルのドキュメントレビューモードへのショートカット。評価軸は issue のドキュメントレビュー評価（[skills/issue/eval.md](../issue/eval.md)）を継承する。ここでは review 特有のショートカット体験と出力品質に関する観点を加える。

## 評価軸（3軸）

### 軸1: 4視点レビューの実行性 — Perspective Execution Score

issue スキルのドキュメントレビューモードを正しく呼び出し、4視点（経営・開発・顧客・ガバナンス）でのレビューが確実に実行されているか。

| スコア | 基準 |
|---|---|
| **3** | issue スキルのフェーズ遷移（Phase 1R → Phase 2 並列 → Phase 3）が完遂され、4視点すべてが具体的に出力されている。 |
| **2** | 4視点は揃っているが、一部視点が一般論に留まる、または Phase 3 クロスカットが薄い。 |
| **1** | 視点欠落 or クロスカット未実行。 |
| **0** | issue スキルを呼び出さず、review 単体で雑な感想を返している。 |

**合格ライン: 2以上**

### 軸2: 指摘の具体度 — Concreteness Score

レビュー指摘が「どの段落・どの主張・どの数字」に対するものか特定され、代替案や改善提案がセットで提示されているか。

| スコア | 基準 |
|---|---|
| **3** | 指摘箇所が引用付きで特定され、代替案・追加すべき論点・削除すべき部分まで明示されている。 |
| **2** | 指摘箇所は特定されているが、代替案が抽象的。 |
| **1** | 「全体的に〜」「もう少し〜」といった抽象的指摘のみ。 |
| **0** | 指摘が感想レベル。 |

**合格ライン: 2以上**

### 軸3: 伝わり方視点 — Deliverability Score

レビュー対象が経営層・GM・ステークホルダー向け文書の場合、読み手の認知負荷・判断しやすさ・「だから何？」の明確さが評価されているか。

| スコア | 基準 |
|---|---|
| **3** | 読み手の意思決定ポイントが明示され、「どこで詰まる／どこを先に読むべき」まで指摘している。3行サマリー・次の一手の有無も評価。 |
| **2** | 伝わり方への言及はあるが、指摘がロジック面に偏り、読み手視点の深さが不足。 |
| **1** | 内容の正しさのみ評価。伝わり方への言及なし。 |
| **0** | レビュー対象の用途（経営報告 / 内部検討 / 外部公開）を識別していない。 |

**合格ライン: 2以上**（経営・GM向け文書をレビューする場合のみ厳格適用）

## 総合

| 対象 | 評価軸 | 合格基準 |
|---|---|---|
| 一般文書レビュー | 軸1 + 軸2 | 両方 ≥ 2 |
| 経営・GM向け文書 | 軸1 + 軸2 + 軸3 | 全て ≥ 2 |
