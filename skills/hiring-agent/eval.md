# hiring-agent スキル評価基準

## 評価軸

### 1. 判定精度（基準忠実性）

スコアリングが `jd.md` / `evaluation-criteria.md` の定義に忠実か。

| レベル | 基準 |
|--------|------|
| A | Must/Want/6軸すべてのスコアに根拠引用があり、基準の行動定義と正確に紐づいている |
| B | スコアに根拠はあるが、一部が基準の行動定義と噛み合っていない |
| C | 一般論（「総合的に」「経験が豊富」）でスコアリングしている。根拠が曖昧 |

**チェック方法:** 出力のスコアを1つ取り、`evaluation-criteria.md` の該当軸 Lv定義と照合する。スコアの根拠発言がそのLvの行動定義に合致するか確認。

### 2. カスタマイズ度（候補者固有性）

面接ガイド・評価が候補者固有の経歴・発言に基づいてカスタマイズされているか。

| レベル | 基準 |
|--------|------|
| A | 書類の弱点に対応する差分Qが正しく差し込まれ、カジュアル面談の結果が技術面接に反映されている |
| B | 差分Qは入っているが、候補者の経歴への言及が薄い（汎用的な追加質問になっている） |
| C | ベーステンプレのコピーになっており、候補者固有のカスタマイズがない |

**チェック方法:** 面接ガイドの★追加Qが、screening.mdのスコア根拠と論理的に対応しているか確認。

### 3. 実用性（そのまま使えるか）

出力がそのまま面接で使える、または判断材料として機能するか。

| レベル | 基準 |
|--------|------|
| A | 面接ガイドを印刷してそのまま面接に持ち込める。評価レポートを見て10秒で判定理由がわかる |
| B | 内容は正しいが、構成が冗長、または重要な情報が埋もれている |
| C | 情報が足りない、または構造が崩れていて使いにくい |

**チェック方法:** 書類選考レポートの「総合判定」→「Mustスコア」→「カジュアル面談ガイド」が上から読むだけで面接準備が完了するか確認。
