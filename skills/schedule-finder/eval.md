# schedule-finder Eval基準

Claude in Chrome 経由で OWA カレンダーを読み取り、空き時間候補を提示するプロンプト生成スキル。評価は「プロンプトの質」と「結果候補の使える度」の両面で見る。

## 評価軸（3軸）

### 軸1: プロンプト生成の適合性 — Prompt Fit Score

入力された条件（対象メンバー・時間枠・期間・制約）がプロンプトテンプレートに正しく埋め込まれ、Claude in Chrome 側が迷わず読み取りに入れる状態になっているか。

| スコア | 基準 |
|---|---|
| **3** | 対象・時間・期間・制約（仮予定の扱い、稼働日ビュー前提等）がすべて反映され、そのまま貼れる完成度。 |
| **2** | 条件は反映されているが、曖昧な指定（「今週中」など）で期間特定が不十分。 |
| **1** | テンプレートをそのまま出しただけで、ユーザー条件の反映が不完全。 |
| **0** | テンプレートを読み込まず独自フォーマットで生成している。 |

**合格ライン: 2以上**

### 軸2: 候補出力フォーマットの揃い — Output Format Score

Claude in Chrome から返ってきた候補を、SKILL.md のテーブル形式（#・日時・状況・備考）に揃え、「おすすめ」を理由付きで提示できているか。

| スコア | 基準 |
|---|---|
| **3** | テーブル形式で揃い、おすすめ候補に理由（移動時間・会議直後の混雑回避・前後予定との兼ね合い等）が明示されている。 |
| **2** | テーブルは揃っているが、おすすめの理由が単なる「早い順」等、選定理由に深みがない。 |
| **1** | フォーマット崩れ or おすすめなし。 |
| **0** | 候補列挙のみで選定判断が放置されている。 |

**合格ライン: 2以上**

### 軸3: 読み取り精度への誠実度 — Accuracy Hygiene Score

OWA の表示状態依存・権限制約（予定あり/空きのみの場合）・スクロール外の予定といった読み取り限界をユーザーに開示し、目視確認を促しているか。

| スコア | 基準 |
|---|---|
| **3** | 読み取れなかった時間帯・スクロール要求・権限制約による推定部分を明示し、「目視確認推奨」まで言い切っている。 |
| **2** | 限界への言及はあるが、ユーザーの判断材料としては不足。 |
| **1** | 限界言及なしで確定的に候補を提示している。 |
| **0** | ハルシネーション（実在しない時間帯・存在しないメンバーの予定）を返している。 |

**合格ライン: 2以上**

## 総合

| 用途 | 評価軸 | 合格基準 |
|---|---|---|
| 通常の空き時間探索 | 軸1 + 軸2 + 軸3 | 全て ≥ 2 |
| 重要会議（役員・外部）の日程調整 | 軸1 + 軸2 + 軸3 | 軸3 = 3（精度開示が必須） |
