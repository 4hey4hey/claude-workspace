# ppt / pptx-creator Eval基準

## 評価軸（3軸）

### 軸1: 設計テンプレ整合性 — Template Conformance Score

生成結果が承認済み設計テンプレと一致しているか。

| スコア | 基準 |
|---|---|
| **3** | 主張文・順序・ボディ要素・図解方式がすべてテンプレ通り。変更ゼロ。 |
| **2** | 軽微な表現変更はあるが、メッセージ方向性は維持。 |
| **1** | 主張文のニュアンスが変わっている、またはスライド順序が変更されている。 |
| **0** | テンプレと大幅に乖離。 |

**合格ライン: 2以上**

### 軸2: 禁止パターン違反ゼロ — Rule Violation Count

ppt-rules.md の禁止パターンに抵触していないか。

| スコア | 基準 |
|---|---|
| **3** | 違反ゼロ。 |
| **2** | 軽微な違反1件（shadow opacity の境界ケース等）。 |
| **1** | 重大な違反1件（暗い背景、英語ラベル、白文字表紙等）。 |
| **0** | 重大な違反2件以上。 |

**合格ライン: 3（ゼロ違反が原則）**

### 軸3: レイアウト多様性 — Layout Variety Score

スライド全体のレイアウトパターンの多様性。

| スコア | 基準 |
|---|---|
| **3** | 3種類以上のレイアウトパターン + 図解3枚以上（10枚以上デッキ時）。 |
| **2** | 3種類以上のレイアウトだが図解が不足。 |
| **1** | 2種類以下のレイアウトパターン。 |
| **0** | 全スライド同じレイアウト。 |

**合格ライン: 2以上（10枚以上のデッキでは3必須）**

## QA自動チェック

pptx-creatorのStep 4（QA）で自動検出可能な項目:
- オーバーフロー検出（python-pptx スクリプト）
- 重なり検出（面積 > 0.1 sq in）
- テキスト抽出（markitdown）による主張文チェック

**QAスクリプトのエラー = 自動不合格**
