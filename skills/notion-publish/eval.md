# notion-publish Eval基準

## 概要

notion-publishスキルの出力品質を測定するための評価基準。
Skill Optimizer が改善提案の効果を検証する際に使用する。

---

## 評価軸（1軸）

### 軸1: 実行信頼性 — Execution Reliability Score

pending状態のoutboxファイルが正しくNotionに公開され、ステータスが更新されるかを評価する。

| スコア | 基準 | 判定ポイント |
|---|---|---|
| **3** | 全pendingファイルが正常に処理。status→published、notion_page_id・notion_url・published_at が全件記録。結果サマリーが正確。 | ゼロエラー完走 |
| **2** | 大半は成功するが、1件のAPIエラー発生。エラーファイルに `status: "error"` が適切にマークされ、他ファイルの処理は継続。 | エラーハンドリングが機能している |
| **1** | 処理は実行されるが、frontmatterの更新漏れ（status が pending のまま）や重複投稿が発生。 | 再実行で二重登録リスク |
| **0** | 処理が開始されない、またはpublished済みファイルを再処理してしまう。 | — |

**合格ライン: 2以上**

**チェック方法:**
1. 処理前: `status: "pending"` のファイルのみが対象か（published/errorはスキップ）
2. 処理後: 各ファイルの `status` が `published` または `error` に更新されているか
3. 処理後: `notion_page_id` と `notion_url` が記録されているか
4. 10件以上の場合: 3秒間隔のレート制限が守られているか
5. 結果サマリーの件数が実際の処理件数と一致するか
