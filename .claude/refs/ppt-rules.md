# PPT生成ルール

## 基本ルール

- `pres.layout` は指定しない（デフォルトの LAYOUT_16x9 = 10×5.625インチを使用）
- 全スライドの background は `{ color: 'FFFFFF' }` に統一（表紙・まとめスライドも含む）
- 座標・サイズはすべて LAYOUT_16x9（幅10インチ × 高さ5.625インチ）基準で設計
- hex カラーに `#` をつけない（PptxGenJS でファイル破損の原因）

## 禁止パターン（AI臭の原因 — 最重要）

> 「やっていいこと」を教えるより「やってはいけないこと」を教える方が出力制御に効く。

### 色・見た目

- **【禁止】暗い背景 + ネオン/シアン系カラー** → 白背景 + 落ち着いた配色を使う
- **【禁止】ドロップシャドウカードの多用** → shadow は opacity 0.08 以下、1スライドに2箇所まで
- **【禁止】スライド全面を覆う大きな色付き矩形** → 部分的なバンド・ライン・カードで表現
- **【禁止】グラデーション背景** → フラットカラーのみ
- **【禁止】大きすぎる角丸**（rectRadius > 0.05）→ 0.03 以下に抑える
- **【絶対禁止】色付き背景に暗い色・黒系の文字を重ねる** → 青背景に黒文字、紺背景に濃紺文字など可読性が低い組み合わせは使わない。色付き（中〜濃色）背景には必ず白文字（`FFFFFF`）を使う。薄色背景（`EEF3FA`, `F2F2F2`, `f5f5f5` 等）のみダーク文字を許容する。draw.io図解にも同じルールを適用する
- **【絶対禁止】表紙スライドを暗い色で覆う** → 以下すべてを遵守すること:
  - `slide.background` は必ず `{ color: 'FFFFFF' }` に固定
  - `addShape` によるカラーバンドは **高さ1.0インチ以下** に制限（スライド高さの18%以下）。`bandH=2.85` のような大面積バンドは絶対禁止
  - タイトル・サブタイトルのテキスト色は **ダーク系**（`theme.P` や `theme.DT`）にする。白文字（`theme.TP = "FFFFFF"`）は表紙では使わない
  - `parts.addCoverTitle` はこのルールを遵守済み。手動で表紙を組む場合も同じ制約を守る

### テキスト・フォント

- **【禁止】英語カテゴリラベルを見出しにする**（"FEATURES"、"CASE STUDIES"、"PRICING" 等）→ 日本語で具体的に書く
- **【禁止】絵文字をビジネススライドに使う** → テキストと図形で表現する
- **【禁止】カテゴリラベルだけのスライドタイトル** → 必ず主張文（assertion）を1文で書く。スライドで言いたいことを言い切る
  - BAD: 「トラフィック構造変化」（ラベル）
  - GOOD: 「AI Overviewにより公益事業のWeb流入が構造的に消失している」（主張）
- **【禁止】1スライドに3種類以上のフォントサイズの見出し** → H2 + H3 の2階層まで
- **【禁止】表紙・まとめスライドの白テキスト** → 白背景上のテキストは必ずダーク系（`theme.P`, `theme.DT` 等）にする。`theme.TP`（白）は表紙タイトルに使わない

### 構造・レイアウト

- **【禁止】全スライド同じレイアウト** → 最低3種類のレイアウトパターンを混在させる
- **【禁止】アイコン+テキストカードだけの構成** → 図解（フロー、マトリクス、比較、ピラミッド、タイムライン）を必ず含める
- **【禁止】テキストだけのスライド** → 図・テーブル・カード等の視覚要素を1つ以上入れる
- **【禁止】タイトル直下のアクセントライン**（AI生成資料の典型パターン）→ ヘッダーセパレータは上部に配置
- **【禁止】「Thank you」「ご清聴ありがとうございました」スライド** → ネクストアクション/まとめで終わる

### 情報設計

- **【禁止】1スライドに主張が2つ以上** → 1スライド1メッセージの原則
- **【禁止】出典なしの数字** → 数値データには必ず出典を添える
- **【禁止】箇条書き5項目以上の羅列** → 4項目以上はカード or テーブルに構造化する
- **【禁止】「〜について」「〜の概要」で終わるタイトル** → 動詞を含む主張文にする

### PptxGenJS 技術制約

- **【絶対禁止】`slide.background` に白以外を設定する** → 原則として全スライド `slide.background = { color: 'FFFFFF' }` のみ使用。表紙の色はバンド形状で表現
  - **例外**: DADSテーマのセクション区切りスライドのみ Blue.700（`264AF4`）を許容。ただしその場合はボトムバー・ヘッダーを省略し、全テキストを白（`FFFFFF`）にすること
- **【禁止】shadow の色に8文字hex（透明度込み）** → `opacity` プロパティを使う
- **【禁止】`bullet` にユニコード文字（"•" 等）** → `bullet: true` を使う
- **【禁止】`option` オブジェクトを複数の `addShape` で使い回す** → PptxGenJS がオブジェクトを破壊的に変換する
- **【禁止】テーブルのセルオプションを共有オブジェクトにする** → 各セルに直接書く

## 必須パターン

- 全スライドにボトムバー（ブランドライン）
- コンテンツスライドにヘッダー（パンくず）+ スライドタイトル（主張文）
- 表紙スライドは白背景ベース（部分的なカラーバンドは可、全面カバーは不可）
- 10枚以上のデッキでは図解スライドを最低3枚含める
- `scripts/lib/` の共有モジュールを使用し、色・フォント・レイアウトをハードコードしない

## DADS テーマ使用時の追加ルール

[デジタル庁デザインシステム（DADS）](https://design.digital.go.jp/dads/) 準拠のテーマ（`dads`, `dads-neutral`）を使用する場合、以下を追加で遵守する。

### カラー

- カラーは `@digital-go-jp/design-tokens` のプリミティブカラー（各色相13階調: 50〜1200）から選択する
- テキストカラーは白背景に対してコントラスト比4.5:1以上を確保する（SolidGray.536 = `767676` が最低ライン）
- 非テキスト要素（罫線・枠線）は白背景に対して3:1以上（SolidGray.420 = `949494` が最低ライン）
- セマンティックカラー（Success / Error / Warning）はDADS公式トークンを使用する

### タイポグラフィ

- フォントは Noto Sans JP を使用する（フォントプリセット `dads`）
- フォントウェイトは Normal（400）と Bold（700）の2種のみ
- 本文サイズ 16CSS px（12pt）以上を基本とする。14CSS px（10.5pt）未満は原則不許可
- 行間は本文150%以上、見出し140%以上

### アクセシビリティ

- WCAG 2.1 AA レベルを目標とする
- 色だけで情報を伝えない（色覚多様性への配慮）
- グラフ・図表の凡例はテキストラベルを併用する
